# Lunil 0.9.1

Lunil 0.9.1 completes the remaining Tier 0 dispatch and call-boundary cleanup tracked by
[#55](https://github.com/dlqw/Lunil/issues/55) and
[#60](https://github.com/dlqw/Lunil/issues/60).

## Tier 0 interpreter

- Charge one remaining-budget unit per interpreter instruction and derive the exact consumed count
  from the entry budget, removing a second hot counter update while preserving accounting across
  interpreter, deoptimization, and generated-code transitions.
- Pass canonical instructions by readonly reference across the scheduler/backend boundary and
  fetch immutable instruction storage without repeated 40-byte value copies.
- Keep backend instruction-accounting validation in Debug builds and remove redundant Release
  overflow checks from instruction-count accumulation; the 64-bit exact-budget limit remains the
  authoritative bound.

## Call and debug hot paths

- Remove repeated call/metamethod argument snapshots. Stack-backed arguments are borrowed until a
  frame, tail-call, yield, or continuation boundary requires one owned snapshot.
- Reuse owned overflow argument packs instead of materializing a second array at runtime call
  boundaries.
- Replace per-query LINQ scans and arrays for active debug locals with a function-version interval
  index built on the cold version-creation path and shared by debug APIs and runtime diagnostics.
- Replace per-call checked register-offset arithmetic with debug assertions over the existing
  verifier and stack-limit invariants.

## Compatibility

- Lua control flow, hook ordering, coroutine/yield behavior, protected calls, close handlers,
  finalizers, exact instruction budgets, public APIs, and package compatibility are unchanged.
- Patch releases on the `0.9.x` compatibility line continue to validate against the reviewed
  `api/0.9.0/` public API and package baselines.
