# Lunil 0.10.0-rc.1

## Release candidate promotion

- Promote the `0.10.0-alpha.2` product to its first release candidate without product-code,
  public-API-baseline, package-scope, canonical-IR, adapter, or evidence-schema changes.
- Freeze the Lua 5.1–5.5 source, generated instruction codec, binary-chunk adapter, runtime,
  standard-library capability, and PUC differential contracts. RC accepts only release-blocking
  fixes; new language, API, IR, generator, and runtime features move to the next milestone.
- Keep Lua 5.4.8 as the unselected/default compatibility baseline and require explicit version
  selection for Lua 5.1, 5.2, 5.3, and 5.5.

## Alpha contract and artifacts

- The release candidate retains Release, public API, package-consumer, trimming, ReadyToRun, CLI
  publish, NativeAOT, conformance, and differential coverage.
- Real PUC differential coverage spans Lua 5.1, 5.2, 5.3, and 5.5; the pinned Lua 5.3 official
  conformance corpus contains 26 scripts, including the original `gc.lua`.
- The immutable `v0.10.0-alpha.2` release produced six RID bundles, 13 NuGet packages, 13 symbol
  packages, SHA-256 asset digests, clean package consumers, and the GitHub prerelease.
- RC validation repeats the same version-derived gates without changing the frozen product scope.
