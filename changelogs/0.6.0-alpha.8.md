# 0.6.0-alpha.8

## Tier 2 数值专门化

- 为稳定的整数、浮点与混合数值 profile 增加真正的 Reflection.Emit Tier 2
  `DynamicMethod`，直接发射 guard、预算检查、ABI v2 寄存器访问、控制流和 deopt。
- 覆盖 Lua 整数溢出、除法、floor division、mod、幂、比较、位运算、移位、浮点运算与
  `2^63` 附近的混合数值比较边界。
- Tier 2 热路径使用 unchecked stack backing-array 访问并保留 GC write barrier；无待关闭变量的
  `Close` 不再强制退出 compiled scheduler。
- `LuaJitTier2Plan.CodeKind` 新增 `ExactNumericSpecializedCil` 诊断；不符合完整专门化条件的
  profile 继续使用 `ManagedProfileProgram`。

## 编译与性能诊断

- 使用 owner-scoped weak cache 在 Tier 1、Tier 2、Loop OSR 与 persisted AOT 之间复用 register
  liveness，不强持有 module owner。
- Tier 2 编译事件新增 IR verification、liveness/cache hit、optimization planning、CIL emission、
  delegate creation、allocated bytes、代码形态、专门化数量与 deopt site 数量。
- 性能脚本新增 `tier2-decision.json` 与六 RID 聚合结果；资格门槛要求 arithmetic paired speedup
  中位数及 bootstrap 95% 下界均至少 4x、allocation slope 接近零、compile p95 小于 10 ms，
  且实际代码形态为 `ExactNumericSpecializedCil`。
- 本地 win-x64 五进程、每进程九个 cold sample 的正式结果为 9.492x 中位加速、
  `[9.124x, 10.018x]` bootstrap 95% interval、6.442 ms compile p95、0 allocation slope 与
  100% liveness cache hit。

## 默认策略与兼容性

- `LuaJitExecutorOptions.Default` 继续只默认启用已通过资格验证的 Tier 1 `Auto`；
  `EnableTier2=false` 与 `EnableLoopOsr=false` 保持不变。
- 精确数值 Tier 2 已可供显式开启的 CoreCLR 宿主使用；managed table/call/metamethod profile 在
  取得更广泛的非回归证据前不会默认启用。
- NativeAOT、trimmed single-file 与 ReadyToRun 发布模式继续通过；动态代码不可用时不会访问
  Reflection.Emit。
