# Lunil 0.12.0-alpha.4

Lunil 0.12.0-alpha.4 adds serialized symbol and function identities that remain usable across
compilation and workspace snapshots.

## Stable symbol keys

- Added `LuaSymbolKey` with canonical, versioned serialization for logical module identity,
  declaration kind, lexical owner, normalized name, and ambiguity ordinal.
- Added semantic-model key generation and resolution for symbols and functions, plus workspace
  overloads accepting `LuaModuleIdentity`.
- Added annotation key generation for class, alias, and enum declarations.
- Keys do not use source offsets or compilation-local symbol/function IDs. Whitespace, comments,
  and unrelated declarations preserve named keys; renames, module changes, and owner changes may
  produce new keys.
