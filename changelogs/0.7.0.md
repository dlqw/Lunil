# Lunil 0.7.0

## Stable compiler, analysis, hosting, and CLI product

- Deliver the complete public compiler pipeline, LuaLS/legacy EmmyLua annotation front end,
  bounded type/control-flow analysis, deterministic incremental module workspace, reusable
  capability-profiled host, MSBuild integration, and packaged `lunil` run/check/build/dump CLI.
- Ship Lua 5.4.8 source and binary-chunk interoperability, the managed runtime and standard library,
  qualified CoreCLR Tier 1/Tier 2 JIT, guarded exact-numeric loop OSR, persisted CIL AOT, and
  build-time NativeAOT integration under one verified execution contract.
- Freeze exact public API baselines for all 14 shipped assemblies and version-independent metadata,
  dependency, normal-asset, symbol-asset, consumer, and tool baselines for all 14 packages.

## Accepted stable evidence

- The unmodified official Lua 5.4.8 user-mode suite reaches `final OK !!!`; six execution backends
  share pinned PUC-Lua observable goldens, and all six release RIDs enforce deterministic fuzz,
  GC/coroutine soak, NativeAOT, package, bundle, and backend evidence gates.
- The accepted `0.7.0-rc.1` source passed the complete six-RID release matrix. Its immutable tag
  published six RID bundles, 14 NuGet packages,
  14 symbol packages, clean local consumers, installed CLI smoke, and the GitHub prerelease.
- Stable `0.7.0` removes the suffix from that accepted candidate without product-code,
  public-API-baseline, or package-scope changes. The stable tag re-runs the same version-derived
  gates and publishes a non-prerelease GitHub Release.
- Make the shared release-version reader treat a missing `VersionSuffix` as the stable SemVer while
  retaining strict validation for both stable and prerelease forms. This allows conformance,
  package, bundle, CI, and release workflows to derive suffix-free `0.7.0` from the same source.

## Compatibility

- `0.7.0` is the first suffix-free release after the superseded `0.6.0-alpha.14` development line;
  no suffix-free `0.6.0` exists.
- Backward-compatible fixes on this line use `0.7.1`. New feature or public-API work begins at
  `0.8.0-alpha.1`; the long-term `1.0.0` compatibility generation remains a later milestone.
