# Lunil 0.6.0-alpha.14

## Release classification

- Publish this version as an alpha prerelease. The qualified JIT/AOT productionization
  milestone is complete, but the full `0.6.0` feature/API scope and long-term support
  contract are not frozen, so beta, RC, and stable labels are not yet appropriate.
- Document the six execution paths and their release defaults in the bilingual README.
- Clarify that `X.Y.Z` selects the compatibility milestone while `alpha`/`beta`/`rc`
  selects maturity; published prerelease numbers are immutable and advance within a channel.
- The release matrix passed all six RIDs, including Release build/test, NativeAOT, bundles,
  packages, publish-mode checks, and aggregated backend evidence.

## Persisted CIL AOT execution

- Add `LuaPersistedAotExecutor`, which executes validated collectible persisted CIL through the
  shared scheduler and exposes `Execute`, `Start`, `Resume`, and `Close` with the same continuation
  semantics as the interpreter, JIT, and static AOT paths.
- Match each closure's canonical module content ID before compiled dispatch. A mismatched module,
  missing function, or disposed artifact deoptimizes at the current canonical PC without repeating
  committed side effects.
- Add `LuaPersistedAotStatistics` for compiled invocations, artifact lookup fallbacks, all compiled
  deoptimizations, expected debug-mode deoptimizations, and unexpected deoptimizations. The caller
  continues to own and dispose `LuaAotLoadedModule` and its collectible load context.

## Loader and performance evidence

- Add `LuaAotLoadMetrics` to attribute validation, assembly loading, delegate binding, total load
  latency, and allocated bytes on successful and failed loads.
- Add `persisted_aot` to the repeated arithmetic, control-flow, Lua-call, table, metamethod, and
  coroutine/error/hook workload matrix. The runner records PE/PDB size, per-phase load p95,
  allocations, compiled invocations, artifact fallbacks, and expected/unexpected deoptimization.
- Add per-RID and six-RID persisted AOT decisions. Qualification requires primary-workload
  throughput confidence bounds, at least 30 warm operations, bounded load latency/allocation and
  artifact size, zero artifact fallback or unexpected deoptimization in every process, explicit
  expected debug-mode deopt attribution, and 0.90 semantic throughput plus 1.10 allocation
  floors/ceilings. Cold-load bounds are 40/25/30 ms for validation/load/binding and 75 ms total.
- Six-process win-x64 evidence qualified at 3.053x arithmetic and 2.578x control-flow medians, with
  a 29.741 ms maximum load p95, 153,368-byte maximum load allocation p95, 19,168-byte largest fixed
  workload artifact, zero allocation slope, and zero artifact fallback or unexpected deoptimization.
- The protected six-RID report qualified every release RID. The aggregate arithmetic and
  control-flow CI95 lower bounds remained at least 2.942x and 2.482x, total-load p95 remained at most
  56.312 ms; every RID's maximum deoptimization count was 1,550,120 expected debug-mode exits.

## Compatibility and tooling

- Preserve `AOT2010` and static-registry behavior when dynamic code is unavailable. NativeAOT still
  uses build-time direct method registration and never loads persisted PE files at runtime.
- Add scheduler, metrics, module-identity, disposed-artifact, and fallback regression coverage.
- Update CI aggregation, compiler design, performance guidance, NativeAOT documentation, bilingual
  README files, and version policy for `0.6.0-alpha.14`.
