# Lunil 0.6.0-alpha.14

## Persisted CIL AOT execution

- Add `LuaPersistedAotExecutor`, which executes validated collectible persisted CIL through the
  shared scheduler and exposes `Execute`, `Start`, `Resume`, and `Close` with the same continuation
  semantics as the interpreter, JIT, and static AOT paths.
- Match each closure's canonical module content ID before compiled dispatch. A mismatched module,
  missing function, or disposed artifact deoptimizes at the current canonical PC without repeating
  committed side effects.
- Add `LuaPersistedAotStatistics` for compiled invocations and interpreter fallbacks. The caller
  continues to own and dispose `LuaAotLoadedModule` and its collectible load context.

## Loader and performance evidence

- Add `LuaAotLoadMetrics` to attribute validation, assembly loading, delegate binding, total load
  latency, and allocated bytes on successful and failed loads.
- Add `persisted_aot` to the repeated arithmetic, control-flow, Lua-call, table, metamethod, and
  coroutine/error/hook workload matrix. The runner records PE/PDB size, per-phase load p95,
  allocations, compiled invocations, and exact fallback counts.
- Add per-RID and six-RID persisted AOT decisions. Qualification requires primary-workload
  throughput confidence bounds, at least 30 warm operations, bounded load latency/allocation and
  artifact size, zero fallback in every process, and 0.90 semantic throughput plus 1.10 allocation
  floors/ceilings.
- Six-process win-x64 evidence qualified at 3.003x arithmetic and 2.428x control-flow medians, with
  a 29.567 ms maximum load p95, 153,368-byte maximum load allocation p95, 19,168-byte largest fixed
  workload artifact, zero allocation slope, and zero interpreter fallbacks.

## Compatibility and tooling

- Preserve `AOT2010` and static-registry behavior when dynamic code is unavailable. NativeAOT still
  uses build-time direct method registration and never loads persisted PE files at runtime.
- Add scheduler, metrics, module-identity, disposed-artifact, and fallback regression coverage.
- Update CI aggregation, compiler design, performance guidance, NativeAOT documentation, bilingual
  README files, and version policy for `0.6.0-alpha.14`.
