# Lunil 0.8.0-alpha.3

## Focused runtime and frontend performance fixes

- Update an existing table entry through the mutation handle returned by the initial raw lookup,
  avoiding the second hash probe in ordinary `__newindex` resolution while preserving key/value
  validation, write barriers, mutation versions, and absent-key metamethod behavior.
- Keep `string.gmatch` source and pattern storage borrowed from their rooted `LuaString` captures
  instead of copying both byte sequences for every iterator call. Store the pattern VM's fixed
  capture workspace inline so matching no longer allocates that scratch array.
- Pre-index bound functions by ID/span and declared symbols by function/span during lowering, and
  index each function's symbols by ID for close-scope analysis. Large nested/captured programs no
  longer rescan complete function or symbol collections at every declaration or closure.
- Track active locals directly in each binder function context and restore the count at scope exit,
  preserving configured Lua local limits without repeatedly counting the global active-symbol
  stack.
- Replace parser `params` terminator arrays with a compact `UInt128` token set and fixed-arity
  overloads, retaining multi-terminator recovery without allocating at every nested block.
