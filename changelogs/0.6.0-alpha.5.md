# 0.6.0-alpha.5

## Backend cache 与 profile

- 新增规范化 `LuaBackendCacheKey`：覆盖 source/canonical/dependency hash、source binding、
  IR/Runtime ABI/codegen/profile/artifact schema、compiler version、优化/debug/hook/sandbox、
  TFM/RID、deployment/trimming 与 feature set。
- 新增跨进程 `LuaBackendDiskCache`：durable temporary write、atomic directory commit、entry/global
  lock、checksum/length 验证、损坏与冲突隔离、entry/quarantine quota、access-marker LRU 和 orphan
  temporary cleanup；权限、lock 和 I/O 故障 fail-soft。
- 新增 deterministic JIT profile v1 导入导出。payload 绑定 canonical module/function/PC/opcode、
  schema/IR/ABI/codegen version 和 footer SHA-256，只保存 owner-free observation；不会持久化
  `LuaState`、closure、table、delegate 或机器码。

## MSBuild、AOT 与恢复

- `Lunil.Build` 默认启用用户级共享 backend cache，并提供 `LunilCacheEnabled`、
  `LunilCacheDirectory` 和 entry/quarantine quota properties。
- `Clean` 删除 `obj/lunil/` 后可从 cache 恢复 canonical IR、persisted CIL 与 Portable PDB；复用前
  重新验证 canonical identity、PE footer/resources/manifest 和 PDB binding。损坏或语义无效
  payload 会先隔离再重新编译。
- `LuaAotArtifactLoader.Validate` 可在不动态加载 assembly 的情况下验证 persisted artifact，适用于
  MSBuild、trimming 与 NativeAOT 流程。

## 可靠性、性能与默认策略

- 新增 release soak 脚本，覆盖跨 backend coroutine/GC、并发 state、tier/code-cache/profile churn、
  AOT/profile truncate/checksum/ABI mismatch 与 MSBuild cache 恢复。
- benchmark runner 现在报告 interpreter、Tier 1、Tier 2 与 Loop OSR 的启动、稳态吞吐、分配、
  编译 p95、RSS、estimated code bytes 和 persisted PE/PDB size；新增多轮汇总脚本。
- 当前 Windows x64 证据未达到 Tier 1/Tier 2 默认启用门槛；Loop OSR 虽有吞吐收益但分配仍偏高。
  因此 `LuaJitExecutorOptions.Default.Policy` 改为 `InterpreterOnly`。`Auto`/`PreferJit`、Tier 2 和
  Loop OSR 继续作为显式 opt-in。
