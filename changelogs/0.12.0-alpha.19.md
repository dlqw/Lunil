# Lunil 0.12.0-alpha.19

## Added

- Added bounded, host-polled `LuaClrTimer` scheduling for game loops with injectable time, explicit
  owner-thread dispatch, and skip, coalesce, or bounded catch-up policies.
- Added the `Timers` CLR capability, Lua `clr.timer`/`clr.cancel_timer` functions, host dispatch APIs,
  and active, pending, quiesced, and stale timer gauges.
- Added runtime-owned `Timer` migration through signed `Continue` and `RejectIfActive` resource rules.

## Reliability

- Module-owned timers now pause with their remaining delay during patch staging. Publication activates
  only candidate callbacks, while execution, migration, barrier, and health rollback restore previous
  schedules and reject candidates. Timer callbacks never enter Lua from a background worker.
