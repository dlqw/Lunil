# Lunil 0.10.1

## Lua 5.1 environment compatibility

- Inject the Lua 5.1 function environment into canonical upvalue zero for binary functions that
  execute `GETGLOBAL`/`SETGLOBAL`, including ancestors that create those closures.
- Preserve ordinary Lua 5.1 upvalue indexes and use the per-closure legacy environment fallback
  for functions without global opcodes.
- Main-chunk converters now identify only upvalue zero as the environment and preserve descriptor
  kinds for additional root upvalues.

## To-be-closed protocol across Lua 5.4 and 5.5

- Centralized the `HasToBeClosedProtocol` version feature for lowering and standard-library
  behavior.
- Lua 5.5 generic-for iterator state and `io.lines` named-file iterators now participate in
  automatic close handling, matching Lua 5.4 semantics.

## Label scope consistency

- Lua 5.3 and later now detect duplicate labels inherited from parent blocks, aligning binder
  diagnostics across all versions that support goto/label syntax.
