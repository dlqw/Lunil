# Lunil 0.10.1

## Patch release

- Publish the reviewed 0.10.x multi-version correctness fixes without changing the supported
  package set, target framework, or compatibility line.
- Preserve the frozen 13-package public API while strengthening official PUC Lua 5.1–5.5
  differential coverage and release qualification.

## Lua 5.1 environment compatibility

- Inject the Lua 5.1 function environment into canonical upvalue zero for binary functions that
  execute `GETGLOBAL`/`SETGLOBAL`, including ancestors that create those closures.
- Preserve ordinary Lua 5.1 upvalue indexes and use the per-closure legacy environment fallback
  for functions without global opcodes.
- Main-chunk converters now identify only upvalue zero as the environment and preserve descriptor
  kinds for additional root upvalues.
- Lua 5.1 canonical chunk writing now removes synthetic `_ENV` upvalues and closure bindings,
  restores `GETGLOBAL`/`SETGLOBAL`, remaps ordinary upvalues, and emits Lua 5.1's absolute
  `LOADNIL` end register so produced chunks execute correctly in PUC Lua 5.1.

## To-be-closed protocol across Lua 5.4 and 5.5

- Centralized the `HasToBeClosedProtocol` version feature for lowering and standard-library
  behavior.
- Lua 5.5 generic-for iterator state and `io.lines` named-file iterators now participate in
  automatic close handling, matching Lua 5.4 semantics.

## Label scope consistency

- Lua 5.4 and 5.5 detect duplicate labels inherited from parent blocks, while Lua 5.2/5.3 retain
  the PUC behavior that permits a nested block to shadow a parent label.

## Version-aware Tier 2 numeric specialization

- Tier 2 JIT optimization and numeric-region planning now consume the module language version.
- Lua 5.1/5.2 stale integer profile hints are normalized to floating-point guards, preventing
  integer arithmetic specialization in versions with a single floating-point number model.
- Loop OSR numeric hints use the same normalization while Lua 5.3 and later retain the integer /
  float numeric model.

## `require` language-version contract

- Documented that source modules loaded by `require` use the current `LuaState` language version;
  cross-version `require` is not supported in the 0.10.x line.
- Added a 0.10.0 migration guide covering separate states for separate Lua version contracts and
  the defensive module/state version check at closure creation.

## Multi-version conformance gates

- Added a required Linux CI job that builds SHA-256-pinned PUC Lua 5.1.5 through 5.5.0 source
  oracles and runs source, imported-chunk, and produced-chunk differential comparisons.
- Added executable Lua 5.3 and 5.4 semantic fixtures with typed assertions for version-specific
  numeric, bitwise, goto, and to-be-closed behavior.
- Documented the distinction between correctness gates and optional cross-runtime performance
  engines; unconfigured third-party engines no longer imply semantic coverage.

## Migration, API, and diagnostic clarity

- Expanded the 0.10.0 migration guide with default/explicit version selection, CLI and API
  examples, Lua 5.1 function environments, chunk-version boundaries, and the 0.7 AOT migration.
- Updated API compatibility and bilingual roadmap documentation for the frozen 0.10.0 contract
  and current conformance claims.
- Refreshed the 0.10.0 public API baseline for the reviewed `HasToBeClosedProtocol` capability
  added by the Lua 5.4/5.5 close-protocol fix.
- Made `LUA0001` version-neutral, corrected `LUA2010` to Lua 5.4-or-later, and report both expected
  and observed version bytes when `Lua54ChunkReader` rejects a chunk.

## Multi-version audit follow-ups

- Replaced Lua 5.3 checks in arithmetic and bitwise hot paths with generated, state-cached
  coercion capabilities while preserving Lua 5.3 numeric-string result tags.
- Split the build-time generators by responsibility and added the missing Lua 5.2 opcode
  contiguity and six-bit-width validation.
- Removed a no-op standard-library branch whose comment incorrectly implied `rawlen` was
  installed conditionally at that location.
