# Lunil 0.8.0-alpha.12

## Breaking: remove Lua AOT

- Remove the persisted/static Lua AOT compiler, PE/PDB artifact and manifest formats, loaders,
  executors, static registry, metadata emitter, disk cache, and their public APIs.
- Remove the `Lunil.Build` project and NuGet package, its `LunilCompile` MSBuild items, generated
  registry, build cache, tests, package assets, and solution entries.
- Remove persisted AOT engines, telemetry, profile-guided artifact rows, and dedicated gates from
  runtime and cross-runtime benchmarks, evidence scripts, soak tests, CI, and release bundles.
- Keep legacy CLI/config/environment AOT targets fail-closed: `--target aot`,
  `buildTarget: "aot"`, and `LUNIL_BUILD_TARGET=aot` return `LUNIL0006`, phase
  `removed-feature`, and exit code `2`. Help and current configuration advertise only `chunk`.
- Retain historical changelogs, performance evidence, ADR 0010, and ADR 0015 as superseded records.
  The active design and migration decision is ADR 0018.

Existing `0.7.x` consumers of Lua AOT are not source compatible with this `0.8` Alpha. Migrate to
runtime source/chunk compilation with `LuaHost`, `LuaInterpreter`, or `LuaJitExecutor`, or distribute
portable PUC Lua 5.4 chunks. The removal is not backported to stable `0.7.x` patches.

## .NET NativeAOT and trimming

- Preserve .NET NativeAOT/trimming support independently of the removed Lua backend. Compiler,
  workspace, CLI, runtime, and interpreter paths remain publishable without dynamic code.
- Change the NativeAOT fixture to compile packaged and dynamic Lua source at runtime, execute it
  through the interpreter, validate workspace analysis, and verify deterministic JIT capability
  fallback without profile collection.
- Document standard SDK `PublishAot`/`PublishTrimmed` consumer publication without `Lunil.Build` or
  a static Lua registry.

## 64-bit generated instruction accounting (#53)

- Add typed CIL-plan `LoadInt64` and `ConvertInt64` instructions, an `Int64Operand`, matching-integer
  arithmetic verification, and Reflection.Emit support for `ldc.i8`/`conv.i8`.
- Widen Tier 1 consumed and frameless-consumed locals, initialization, accumulation, frameless-call
  merge, deoptimization, and compiled-exit call targets to `Int64`/`long`.
- Bind Tier 2 generated exits and frameless merges to the `long` runtime ABI while retaining the
  existing frameless-call helper's `int` result and explicitly widening it at the boundary.
- Add a non-billion-iteration Reflection.Emit regression proving `Int32.MaxValue + 10` survives the
  generated exit unchanged.

## Verification

- The full Release solution passed all 834 tests, including 165 code-generation tests, 214 runtime
  tests, 46 CLI tests, 16 five-backend differential tests, conformance, and stability coverage.
- CLI removed-option tests cover command-line, JSON configuration, and environment-variable input.
- All 13 reviewed public API baselines and 13 NuGet package baselines passed exact validation;
  local package consumers and the `Lunil.Cli` tool package passed smoke tests.
- The win-x64 .NET NativeAOT fixture and CLI passed publication and execution. Trimmed single-file
  and ReadyToRun publication modes also passed, as did the win-x64 release bundle.
- The complete win-x64 250 ms, six-round cross-runtime gate passed: all 384 correctness-checked samples completed
  without instruction-counter overflow, and all 16 Auto/Tier 2 workload gates passed. Six-RID CI
  evidence remains the cross-platform release qualification source.
