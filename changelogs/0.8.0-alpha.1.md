# Lunil 0.8.0-alpha.1

## String-heavy standard-library workloads

- Make `table.concat` and `string.gsub` build their result in checked linear byte buffers instead
  of materializing every growing prefix as a logical Lua string. The original 50,000-element
  concat and 500,000-match substitution workloads now complete under a 16 MiB logical heap.
- Preserve the linear buffer across Lua function and table/metamethod callbacks with an
  owner-validated byte-only native continuation state. Lua values remain in the existing
  GC-visible continuation window, including during full collections and coroutine transitions.
- Keep binary strings, captures, empty matches, replacement templates, non-yieldable callback
  boundaries, and table `__len`/`__index` ordering intact. Reuse a span-based pattern matcher and
  write template captures directly without transient logical strings.
- Add deterministic N/2N allocation tests, the issue reproductions, quota-failure coverage,
  callback/GC/ownership tests, and runtime benchmark evidence for concat and substitution growth.

## JIT fallback routing

- Route cold and terminally rejected functions through the shared reference interpreter instead of
  returning a zero-instruction deoptimization for every canonical instruction. Function decisions
  are cached per function behind weak module keys, preserving mixed-module behavior without
  retaining modules, states, or closures.
- Admit small functions once invocation hotness proves repeated execution, so recursive and other
  call-hot functions are no longer rejected solely because they have no control-flow backedge.
- Emit fallback telemetry only for bounded function-state transitions and avoid constructing a
  fallback event when there are no subscribers.
- Add recursive Fibonacci and table-heavy sieve backend evidence, including fallback allocation and
  event counts, and enforce interpreter-parity gates for rejected workloads.

## Tier 2 numeric loops and profiling

- Emit `NumericForPrepare`, `NumericForLoop`, and vararg setup directly in exact-numeric Tier 2 and
  continue canonical PC dispatch inside the same generated method, eliminating per-iteration
  scheduler exits.
- Require fail-closed native coverage for the complete exact-numeric CFG and report unsupported
  shapes as `JIT2106` before compilation instead of installing a partially specialized slow-path
  method.
- Keep profiled and plain Tier 1 delegates in the bounded code cache and switch terminal Tier 2
  rejections to the plain delegate, removing permanent per-instruction observation overhead while
  preserving invalidation, import, eviction, concurrency, and guard-failure reprofiling behavior;
  no-hotspot profiles receive one larger retry window before the same bounded terminal transition.
- Split Tier 2 telemetry into method entries, completed invocations, and unsupported exits; extend
  correctness/performance coverage with numeric-for edges, iterative Fibonacci, Mandelbrot,
  polymorphic profiles, instruction budgets, debug hooks, stress GC, and invalidation.
