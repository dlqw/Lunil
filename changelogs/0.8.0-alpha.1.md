# Lunil 0.8.0-alpha.1

## Cross-runtime performance evidence

- Add a common-subset workload suite that uses pinned native Lua 5.4.8 as the per-RID baseline and
  compares LuaJIT, MoonSharp, and Lunil interpreter, Auto, Tier 1, Tier 2, Loop OSR, and persisted
  AOT configurations. Every row validates its result and records the actual Lunil execution route.
- Build native Lua and LuaJIT from checksum-verified pinned sources instead of relying on ambient
  executables. Record source, runtime, workload, commit, and executable hashes with every run.
- Calibrate process-CPU batches independently, reset managed heaps outside the timed region, rotate
  engine order over six rounds, emit raw and summarized JSON/CSV plus Markdown, and compute
  deterministic bootstrap intervals from matched round-level ratios rather than independent
  medians.
- Gate Auto and Tier 2 on every workload at a median of at least 1.05x versus MoonSharp with a
  paired CI95 lower bound of at least 1.00x, a stable compiled route, and clean timed-region
  fallback/deoptimization telemetry. The six-RID aggregator fails closed on any missing,
  duplicate, incomplete, schema-incompatible, or failed RID report.

## Table, call, and string hot paths

- Read and update proven dense integer table slots without generic key normalization, while
  preserving owner validation, write barriers, logical quota, array/hash-tail migration, and
  mutation/storage versions. Pool physical array storage and reuse bounded allocation-site
  capacity feedback without changing Lua-visible table length or logical accounting.
- Keep eligible small leaf Lua calls frameless and continue successful calls inside the same Tier 1
  or guarded Tier 2 method. Exact instruction budgets, multi-result windows, debug hooks, GC roots,
  finalizers, close/unwind state, and guard fallback retain canonical behavior.
- Avoid repeated numeric conversion after a primitive guard has established numeric kinds, cache
  bounded short string/integer concatenations weakly, and make raw `table.concat` use dense reads,
  paired appends, UTF-8 number formatting, and pooled long-string buffers.
- The final Windows x64 Release matrix completed all 432 samples. Auto/Tier 2 beat MoonSharp on all
  eight workloads, with 1.65x/1.69x geometric means and per-workload paired medians ranging from
  1.23x/1.23x (`string_build`) to 2.39x/2.37x (`arithmetic`).
- Add bounded logical-GC polls to generic compiled backedges, materialize the exact post-instruction
  GC map and resume PC before finalizer dispatch, and clear physical Lua stack roots eliminated by
  Tier 2 dead-move optimization.
- Qualify the complete six-RID hosted run: all 96 Auto/Tier 2 workload gates passed, with aggregate
  MoonSharp-relative geometric means of 1.980x/1.979x and a minimum paired median of 1.067x/1.054x.

## Lua 5.4 chunk generation and canonical lowering

- Select Lua 5.4 immediate and constant opcodes, fuse comparison branches, remove redundant
  moves and boolean materialization, and shorten temporary-register lifetimes while preserving
  exact source/debug boundaries. Canonical constants are deduplicated during conversion.
- Preserve Lua 5.4 edge semantics for flipped metamethod comparisons, floating-point bitwise
  constants, aliased comparison operands, captured parameters/registers, close ranges, and caller
  live slots across runtime operations. Official PUC Lua 5.4.8 conformance remains the gate.

## Hosting and CLI execution backends

- Make `LuaHost` select `Auto`, `Interpreter`, or `Jit` execution explicitly. `Auto` uses the
  qualified tiered JIT when .NET dynamic code is available and falls back to the reference
  interpreter on NativeAOT or other dynamic-code-disabled runtimes; rejected functions retain the
  same semantic interpreter route and execution budgets.
- Expose the requested backend and JIT options through `LuaHostOptions`, and report dynamic-code
  availability, the selected concrete backend, and lazily initialized `LuaJitStatistics` from the
  host. `Lunil.Hosting` now carries the corresponding `Lunil.CodeGen.Cil` package dependency.
- Add CLI `--execution auto|interpreter|jit`, JSON `execution`, and `LUNIL_EXECUTION` selection.
  Check/build/dump reuse a direct workspace service instead of constructing an execution host,
  reducing cold-start initialization while keeping diagnostics and module resolution unchanged.

## String-heavy standard-library workloads

- Make `table.concat` and `string.gsub` build their result in checked linear byte buffers instead
  of materializing every growing prefix as a logical Lua string. The original 50,000-element
  concat and 500,000-match substitution workloads now complete under a 16 MiB logical heap.
- Preserve the linear buffer across Lua function and table/metamethod callbacks with an
  owner-validated byte-only native continuation state. Lua values remain in the existing
  GC-visible continuation window, including during full collections and coroutine transitions.
- Run resumable native descriptors correctly when they are selected as arithmetic, indexing, or
  concatenation metamethods, preserving the caller's live register window, result transform, and
  non-yieldable callback boundary across Lua callbacks or coroutine suspension.
- Keep binary strings, captures, empty matches, replacement templates, non-yieldable callback
  boundaries, and table `__len`/`__index` ordering intact. Reuse a span-based pattern matcher and
  write template captures directly without transient logical strings.
- Add deterministic N/2N allocation tests, the issue reproductions, quota-failure coverage,
  callback/GC/ownership tests, and runtime benchmark evidence for concat and substitution growth.

## JIT fallback routing

- Route cold and terminally rejected functions through the shared reference interpreter instead of
  returning a zero-instruction deoptimization for every canonical instruction. Function decisions
  are cached per function behind weak module keys, preserving mixed-module behavior without
  retaining modules, states, or closures.
- Admit small functions once invocation hotness proves repeated execution, so recursive and other
  call-hot functions are no longer rejected solely because they have no control-flow backedge.
- Emit fallback telemetry only for bounded function-state transitions and avoid constructing a
  fallback event when there are no subscribers.
- Add recursive Fibonacci and table-heavy sieve backend evidence, including fallback allocation and
  event counts, and enforce interpreter-parity gates for rejected workloads.

## Tier 2 numeric loops and profiling

- Emit `NumericForPrepare`, `NumericForLoop`, and vararg setup directly in exact-numeric Tier 2 and
  continue canonical PC dispatch inside the same generated method, eliminating per-iteration
  scheduler exits.
- Require fail-closed native coverage for the complete exact-numeric CFG and report unsupported
  shapes as `JIT2106` before compilation instead of installing a partially specialized slow-path
  method.
- Keep profiled and plain Tier 1 delegates in the bounded code cache and switch terminal Tier 2
  rejections to the plain delegate, removing permanent per-instruction observation overhead while
  preserving invalidation, import, eviction, concurrency, and guard-failure reprofiling behavior;
  no-hotspot profiles receive one larger retry window before the same bounded terminal transition.
- Split Tier 2 telemetry into method entries, completed invocations, and unsupported exits; extend
  correctness/performance coverage with numeric-for edges, iterative Fibonacci, Mandelbrot,
  polymorphic profiles, instruction budgets, debug hooks, stress GC, and invalidation.
- Share one reducible-CFG numeric-region planner and emitter between exact-numeric Tier 2 and Loop
  OSR. Versioned unboxed locals, direct numeric CIL, multi-latch natural loops, side exits,
  `SetTop`, captured live-outs, mixed numeric values, integer wrap, shifts, NaN, and negative zero
  retain canonical behavior.
- Replace hot-loop per-instruction budget comparisons with a proved safepoint quantum and
  basic-block accounting. A separate cold slow tail preserves every exact budget PC,
  budget-before-tag-guard ordering, and failing `//`/`%` instruction rollback; the new plan telemetry
  requires zero hot instruction-budget checks while retaining nonzero region, unboxed-local,
  direct-instruction, and safepoint facts.
- Materialize all GC-visible state at most every 1,024 backedges before polling. An already acquired
  Tier 2/Loop OSR delegate may finish across invalidation, but later dispatch cannot re-enter it and
  asynchronous results compiled across invalidate, clear, or dispose cannot be published.
