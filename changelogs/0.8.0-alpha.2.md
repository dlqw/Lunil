# Lunil 0.8.0-alpha.2

## Performance issue baselines

- Add a `--micro-only` runtime benchmark mode so focused local evidence does not also execute the
  cross-backend qualification matrix.
- Record generational minor-collection scaling against a retained old heap. The workload promotes
  the retained graph before timing fixed young-generation churn, exposing old-heap traversal and
  temporary-allocation growth independently of setup.
- Add focused allocation and time evidence for `table.insert`/`table.remove`, `table.unpack`, and
  `string.gmatch` workloads that exercise the remaining standard-library findings from #42.
- Measure frontend binding/lowering growth with generated captured locals and nested functions,
  providing a repeatable baseline for symbol-resolution and activation-index changes.
