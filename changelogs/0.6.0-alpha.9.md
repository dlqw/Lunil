# 0.6.0-alpha.9

## Tier 2 默认自动晋升

- `LuaJitExecutorOptions.EnableTier2` 默认改为 `true`。CoreCLR release 默认会在 Tier 1 变热后，
  自动晋升已通过资格检查的精确整数、浮点与混合数值 hotspot。
- 新增 `EnableTier2ManagedFallback`，默认 `false`。table、upvalue、closure、call/tail-call、
  to-be-closed 等 managed semantic profile 继续留在 Tier 1；宿主只有显式开启该选项才会恢复
  实验性的 `ManagedProfileProgram`。
- `EnableTier2=false` 仍是完整 opt-out，同时关闭 profile 采集、profile 导入、promotion 与 Tier 2
  compiler 预热。

## 确定性资格与防绕过

- 新增 `LuaJitTier2Eligibility`、`GetTier2PromotionEligibility`、
  `Tier2EligibilityAccepted/Rejected` 事件、独立统计及 `JIT2101`–`JIT2105` 诊断。
- 自动资格复用生产 Tier 2 optimization/liveness planner；只有可保证生成
  `ExactNumericSpecializedCil`、至少包含一个 exact numeric specialization 且没有 managed semantic
  site 的 profile 才会进入 compile queue。
- polymorphic/managed 拒绝永久缓存；尚无数值 hotspot 的 profile 使用指数 sample backoff 复查。
  profile import 与 guard failure 后的扩宽 profile 必须重新通过同一门控。
- 安装阶段再次验证实际 `LuaJitTier2Plan.CodeKind`；自定义 compiler 不能把
  `ManagedProfileProgram` 发布到默认自动路径。
- Tier 2 逐指令 profile 采集只在 Tier 1 安装后激活；被 Tier 1 `Auto` 收益检查拒绝的函数不会
  承担默认 rollout 的 profile observation 成本。

## 兼容性、测试与性能

- NativeAOT 或其他动态代码不可用环境不采集 Tier 2 profile，profile import 返回 `Disabled`，并
  继续使用 interpreter 或构建期 AOT。
- 新增默认配置、exact acceptance、table/managed-boundary rejection、polymorphic profile、
  imported profile、guard widening、unexpected code kind、并发 promotion 与 NativeAOT 回退测试。
- backend evidence runner 改为直接测量 release 默认 Tier 2，并记录 eligibility 与 managed
  installation count；负收益矩阵若自动安装任何 managed Tier 2 方法，或 paired
  Tier 2-enabled/Tier 2-disabled `Auto` median 低于 0.90，即判定失败。
- win-x64 五进程正式结果为 11.282x arithmetic median speedup、`[8.754x, 11.864x]` bootstrap
  95% interval、0.234 ms Tier 2 compile p95、零 allocation slope、100% liveness cache hit；
  lua-call/table/metamethod/coroutine-error-hook 矩阵自动安装的 managed Tier 2 方法数为 0，
  paired default-on/default-off median 最低为 0.981x。
- Loop OSR 默认值保持 `false`。
