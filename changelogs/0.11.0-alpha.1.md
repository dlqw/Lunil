# Lunil 0.11.0-alpha.1

This prerelease opens the 0.11 compatibility line with opt-in CLR type discovery and object
construction for embedding hosts.

## CLR interoperation

- Add `LuaClrOptions`, `LuaClrCapabilities`, and `LuaClrBridge` to configure exact assembly and type
  allowlists without loading assemblies by name.
- Add deterministic public-constructor selection for Lua nil, boolean, string, integer, float,
  enum, nullable, `LuaValue`, object, and compatible CLR-userdata arguments.
- Add the optional Lua `clr.type` and `clr.new` functions. The bridge is disabled and absent from
  Lua globals by default.
- Represent constructed objects as host-owned userdata through `LuaClrObject`; owned
  `IDisposable` instances are released at most once.
- Expose stable `LuaClrErrorCode` values to embedding applications and catchable error messages to
  Lua callers.

## Compatibility

- Existing hosts retain their previous behavior unless `LuaHostOptions.Clr` is configured.
- CLR discovery considers public types in already loaded, exact-allowlisted assemblies only.
- Interpreter and dynamic-JIT host paths share the same bridge behavior.

See the [CLR interoperation guide](../docs/clr-interop.md) and
[0.11 migration guide](../docs/migration-0.11.0.md).
