# Lunil 0.7.0-alpha.2

## Annotation front end

- Add the `Lunil.EmmyLua` package with a shared bounded annotation lexer and public immutable
  syntax models for named, literal, union, intersection, nullable, array, tuple/type-pack,
  vararg, function, generic, and structural-table types.
- Add independent `LuaLsAnnotationParser` and `LegacyEmmyAnnotationParser` entry points. LuaLS
  remains the default; compatible mode selects the legacy parser only when it produces a better
  bounded parse and records that decision as an informational diagnostic.
- Parse class, field, alias and alias-continuation, enum, type, param, return, generic, overload,
  vararg, cast, diagnostic, operator, marker, and unknown directives while preserving unknown
  tags and their payloads instead of discarding extension data.
- Add configurable annotation, token, type-depth, and diagnostic budgets. Annotation syntax
  diagnostics default to warnings and can be promoted to errors by analysis-oriented hosts.
- Implement global diagnostic-code suppression plus source `disable`, `enable`, and
  `disable-next-line` directives without changing Lua runtime semantics.

## Compiler integration

- Add annotation options and an annotation phase to `LuaCompiler`; immutable annotation results
  are published beside syntax and semantic results before lowering.
- Keep annotations erased from canonical IR and runtime execution. Default annotation warnings
  do not block source execution, while explicitly promoted annotation errors prevent module
  publication through the existing compiler success contract.
- Add annotation package and test projects to `Lunil.sln`; automatic package and release-bundle
  discovery includes `Lunil.EmmyLua`.

## Tests and documentation

- Cover LuaLS directive/type parsing, class generics, structural/function types, legacy fallback,
  unknown-tag preservation, suppression, configured budgets, deterministic random-byte parser
  fuzzing, compiler integration, and runtime erasure.
- Update the roadmap, compiler design, version policy, and bilingual product status for the
  second `0.7.0` alpha milestone.
