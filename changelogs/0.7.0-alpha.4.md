# Lunil 0.7.0-alpha.4

## Module graph and resolution

- Add the public `Lunil.Workspace` package with stable logical module/source identities, immutable
  documents and snapshots, injectable resolvers, an in-memory resolver, and a root-confined file
  resolver with Lua-style path patterns.
- Extract direct global literal `require` calls without misclassifying shadowed locals. Record
  unresolved static imports and dynamic require names as stable diagnostics and conservative graph
  boundaries without changing runtime package-loader behavior.
- Publish deterministic nodes, edges, strongly connected components, source/content identities,
  module results, export types, cache/fixed-point metrics, and phase-attributed `LUA7xxx` diagnostics.

## Incremental cross-module analysis

- Feed resolved dependency export types into `Lunil.Analysis`, infer module exports from top-level
  return packs, and process the condensation graph dependency-first. Cyclic SCCs use bounded,
  deterministic fixed points and stable widening when their configured iteration limit is reached.
- Add reusable SHA-256-addressed discovery and analysis caches. Analysis keys include direct
  dependency export hashes so a leaf implementation change with the same public export reanalyzes
  only that leaf, while an export change minimally invalidates its dependents.
- Add global module, source-byte, dependency, cache-entry, fixed-point, diagnostic, and parallelism
  budgets; serialize snapshots per workspace, honor cancellation without partial publication, and
  merge all modules and diagnostics deterministically under globally bounded scheduling. Disposal
  lets snapshots already in flight finish safely while rejecting new operations.

## Product integration and verification

- Extend `LuaCompiler` with immutable external analysis environments and resolved module types;
  builtin handling now respects lexical shadowing. Keep workspace/type metadata erased from
  canonical IR and runtime behavior.
- Expose a reusable workspace through `LuaHost`, preflight all source `LunilCompile` items through
  the same graph before persisted AOT emission, and include workspace analysis in NativeAOT,
  trimming, ReadyToRun, NuGet, and release-bundle coverage.
- Cover static/dynamic/shadowed require handling, file resolution, cycles and widening, dependency
  type propagation, minimal invalidation, cache hits, budgets, cancellation, deterministic bounded
  parallel merging, Hosting/MSBuild integration, and NativeAOT execution.
