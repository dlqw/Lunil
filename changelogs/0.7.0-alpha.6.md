# Lunil 0.7.0-alpha.6

## Lua 5.4.8 conformance

- Vendor the unmodified official Lua 5.4.8 test archive and all 41 extracted fixtures under a
  dedicated conformance project. Pin the archive and every fixture by SHA-256, classify the
  managed user-mode contract, and run the unchanged `all.lua` driver with `_U=true` through
  `LuaHost` until it reports `final OK !!!`.
- Align protected and unprotected errors, reader-based `load`, parser diagnostics, main-chunk
  `_ENV`, tail calls, generic-for closing, to-be-closed locals, weak tables, finalizers,
  coroutine transitions, and garbage-collector barriers with the PUC-Lua 5.4.8 observable
  behavior exercised by the official suite.
- Complete the debug surface needed by the suite: hook call/return/line/count/tail-call ordering,
  transferred argument/result ranges, native/metamethod/close/finalizer/iterator frame naming,
  local and vararg inspection/mutation, stripped-function metadata, active lines, suspended
  coroutine tracebacks, and PUC-compatible deep-stack truncation.
- Fix integer-boundary behavior in `table.unpack`, proxy and type validation in `table.move`,
  `type` versus metatable `__name`, portable locale reporting, empty reader termination, and
  Lua-compatible syntax-error context. Large and open table constructors now reuse temporary
  registers without exceeding the Lua 5.4 register limit.
- Preserve Lua 5.4 binary-chunk semantics for plain jumps, missing upvalue debug names, open
  captured locals, 200 upvalues, finalizer-visible writes, long-string constant identity, and
  converter scratch registers. Correct canonical `SETLIST` register liveness across persisted
  CIL AOT and JIT backends.

## Differential and stability gates

- Add eight pinned PUC-Lua 5.4.8 observable goldens covering exact debug-hook traces, protected
  errors, close/finalizer order, `coroutine.close`, integer limits, a 300-field open `SETLIST`,
  binary `string.dump`/`load` round trips, and table-library boundary cases. Every golden runs
  unconditionally on the interpreter, executor-auto, persisted CIL AOT, Tier 1, Tier 2, and Loop
  OSR backends.
- Add a provenance-checked golden updater that requires an explicitly trusted executable
  SHA-256, `Lua 5.4.8` executable version output, `_VERSION == "Lua 5.4"`, and the pinned official
  source archive before replacing committed observations.
- Add deterministic, budgeted, replayable fuzz gates for syntax, Lua 5.4 chunks, annotations, and
  type analysis. Failures report the fixed seed, case index, and Base64 input. Add a fixed-seed,
  stress-every-allocation GC/coroutine/finalizer soak with explicit heap, stack, call-depth, and
  instruction budgets.
- Ensure malformed table constructors always advance after an expression terminator instead of
  looping and growing memory indefinitely; retain the discovered fuzz input as a focused parser
  regression.

## Six-RID release evidence

- Produce per-RID JSON evidence from TRX results for the official suite, all six differential
  backends, and deterministic stability gates. Evidence records the Git commit, runtime identity,
  fixture/manifest/golden hashes, oracle provenance, backend catalog, seeds, case counts, and
  budgets.
- Extend CI across Windows, Linux, and macOS on x64 and Arm64 to upload the evidence and require a
  single aggregate with all six RIDs on the same commit and pinned contract. NativeAOT, bundles,
  package validation, backend performance evidence, and the existing publish-mode gates remain
  mandatory.
