# 0.6.0-alpha.4

## 新增

- 新增 `Lunil.Build` NuGet package：`build`/`buildTransitive` props/targets、`LunilCompile`
  item contract 和 bundled net10.0 MSBuild task dependencies。
- source 与 PUC Lua 5.4 binary chunk 可在宿主编译前完成 frontend/chunk verification、canonical IR
  lowering 和 persisted CIL/PDB emission；artifact、canonical module、manifest 与 generated registry
  输出到按 configuration/TFM/RID 隔离的 `obj/lunil/`。
- 新增 `LuaStaticAotRegistry`、`LuaStaticAotModule` 与 `LuaStaticAotExecutor`。generated `.g.cs`
  通过 `ModuleInitializer` 和直接 method group 注册 function/shard，并内嵌 verified canonical module，
  不依赖 runtime reflection/discovery。

## NativeAOT 与构建语义

- NativeAOT 中 JIT 自动禁用；静态模块命中预编译入口，动态模块在当前 canonical PC 精确回退
  interpreter；动态 PE loader 在 dynamic code unavailable 时返回 `AOT2010`。
- 增量 build 以 source hash、input kind、optimization/debug/sandbox、TFM 与 RID 复用 artifact，
  同时在每次 MSBuild 中恢复 generated source/reference item；原子写、output lock、design-time 隔离和
  `Clean` 语义覆盖 clean/rebuild/incremental/parallel build。
- 六 RID CI 新增同源 NativeAOT publish/run corpus；`win-x64` 额外验证 trimmed self-contained
  single-file 与 ReadyToRun/CoreCLR。

## 诊断与兼容

- 新增 `LUNIL1001`–`LUNIL1008`、`LUNIL2001`–`LUNIL2003` 与 `LUNIL9001` build diagnostic；
  Lua source error 保留一基 line/UTF-16 column。
- 移除 resumable native delegate capture 检查中的 runtime reflection，明确要求静态方法组入口，
  并为 JIT/动态 AOT loader 标注准确的 trim/NativeAOT contract，使 fixture 在 trimming/AOT
  analyzer 下零 warning。
