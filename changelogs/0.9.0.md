# Lunil 0.9.0

Lunil 0.9.0 is the stable release of the shared Tier 2 string and numeric execution work.
The release is built from the reviewed Alpha 5/Beta-qualified source without product-code changes
after the accepted qualification commit.

## Highlights

- Keeps stable string-number and string-float concatenation, string length, and dense string-array
  writes inside one guarded Tier 2 region.
- Preserves operand order, table guards, write barriers, exact instruction budgets, write-before-
  deopt ordering, binary-string semantics, and canonical restart behavior.
- Retains the existing dense `table.concat` implementation. Segmented diagnostic evidence showed
  that a separate bulk-copy path was not a release bottleneck.
- Continues to support the reference interpreter, Tier 1, Tier 2, loop-entry OSR, NativeAOT,
  trimming, single-file, and ReadyToRun deployment modes.

## Qualification

- Auto JIT geometric mean: `1.688x` native Lua and `24.089x` MoonSharp.
- Lowest per-RID Auto geometric mean: `1.103x` native Lua.
- `control_flow`: `2.101x`; `string_build`: `2.164x`.
- Six release RIDs passed the Lua 5.4.8 conformance, five-backend differential, deterministic
  fuzz/GC, weak-table/finalizer, NativeAOT, trimming, package, CLI, public API, and release-bundle
  gates.
- Tier 2 and Loop OSR compilation, allocation, startup, code-size, and unchanged-route regression
  gates passed the `0.9.0` roadmap limits.

The reproducible qualification data is available in the
[0.9.0 performance report](../benchmarks/results/0.9.0-performance.json) and
[performance documentation](../docs/performance.md).

## Compatibility

The stable `0.9.0` line targets .NET 10 and Lua 5.4.8. Native Lua C modules remain unsupported
because Lunil does not expose the Lua C ABI.
