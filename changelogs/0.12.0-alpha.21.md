# Lunil 0.12.0-alpha.21

## Added

- Added a cross-process prepared/health barrier for `LuaPatchCoordinator`, with pinned participant
  membership, canonical manifest identity, quorum selection, immutable commit/rollback decisions,
  and bounded polling deadlines.
- Added `ILuaPatchDistributedBarrierStore` for external control planes and
  `LuaPatchFileDistributedBarrierStore` for processes sharing a lock-correct file system.
- Added explicit pruning and hard bounds for file-backed barrier state, participants, identities,
  messages, and abandoned temporary or lock files.

## Reliability

- Selected processes retain rollback-capable sessions and replay reservations until every selected
  participant acknowledges health. A peer failure, timeout, configuration conflict, corrupt state,
  or store failure now fails closed without accepting a partial ring.
- Durable barrier snapshots use operating-system locks, flushed atomic replacement, canonical
  manifest SHA-256 pinning, structural validation, and corruption hashes. Once a global commit is
  durable, a later local journal or traffic-restoration failure no longer causes a split-brain local
  rollback.
