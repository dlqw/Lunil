# Lunil 0.7.0-alpha.5

## Command-line product

- Add the packaged `Lunil.Cli` .NET tool and RID apphost with `lunil run`, `lunil check`,
  `lunil build`, and `lunil dump`, plus command-aware help, assembly informational version output,
  Ctrl+C cancellation, and stable success/diagnostic/usage/input/execution/build/cancel exit codes.
- Keep program stdout separate from diagnostic stderr. Support human-readable and stable
  `lunil.diagnostics.v1` JSON diagnostics, warnings-as-errors, source stdin, main-chunk varargs, and
  the global `arg` table.
- Add bounded UTF-8 response files with quoting, Windows-safe backslash handling, comments,
  literal `@@`, nested relative paths, cycle detection, and depth/file-size/argument-count limits.
  Add strict `lunil.json` discovery/selection and `LUNIL_*` environment overrides with precedence
  defaults < configuration < environment < CLI.

## Workspace, sandbox, and artifacts

- Preflight source execution, checking, building, and dumping through `Lunil.Workspace`; derive
  stable module/source identities, resolve Lua path patterns, propagate cross-module analysis, and
  configure runtime `package.path` without changing package-loader semantics.
- Expose trusted, root-confined read-only sandbox, and deterministic profiles. Sandbox reads reject
  traversal outside roots and symlink/reparse-point traversal; writes and temporary files are
  denied. All profiles honor input, instruction, stack-slot, call-depth, and logical-heap budgets.
- Build canonical PUC Lua 5.4 chunks for one module or a resolved workspace. Build persisted CIL AOT
  assemblies with optional portable PDBs, canonical module payloads, and versioned JSON manifests.
  Dump summary, syntax, annotation, analysis, IR, and chunk views as text or `lunil.dump.v1` JSON.
- Reject command-specific options outside their owning command instead of silently accepting them,
  and translate Lua runtime/budget failures into execution diagnostics rather than host exceptions.

## Distribution and verification

- Add focused CLI coverage for source/chunk execution, script arguments, workspace requires,
  sandbox denial, deterministic clocks, runtime budgets, configuration precedence, response files,
  JSON diagnostics, chunk/AOT builds, every dump view, malformed/missing/oversized inputs, option
  validation, and cancellation.
- Include `lunil` apphost/runtime metadata in all six RID bundles and package `Lunil.Cli` alongside
  the component packages. CI installs the locally built tool package and runs source/chunk/dump
  smoke tests.
- Publish and execute the CLI under NativeAOT on all six release RIDs, and under trimmed
  single-file and ReadyToRun modes on the publish-mode gate, covering response/config parsing,
  deterministic execution, JSON diagnostics, and chunk building.
