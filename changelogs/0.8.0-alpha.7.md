# Lunil 0.8.0-alpha.7

## Persisted profile-guided numeric AOT

- Add `LuaAotCompilationOptions.Profile` so an exact-module `LuaJitModuleProfile` can drive
  persisted numeric-region specialization. The stable serialized profile fingerprint, policy
  version, and profile-enabled flag participate in the options fingerprint and assembly identity.
- Reject a profile for another canonical module as `AOT1005`; reject malformed, incomplete,
  duplicate, or incompatible profile data as `AOT1006`. Loader validation also rejects corrupted
  profile policy, fingerprints, region topology, metrics, or method bindings before assembly load.
- Share one backend-neutral numeric-region IL generator between Reflection.Emit Tier 2 and the
  metadata PE emitter, avoiding semantic drift across dynamic and persisted code generation.
- Emit persisted numeric methods with real unboxed `long`, `double`, and `bool` locals, guarded
  mixed-numeric operations, direct control flow, precise deoptimization, budget accounting, and
  safepoint polling. Portable PDB method rows now cover these generated methods as well.
- Split canonical fallback shards at numeric-region entry and exit boundaries. The dispatcher can
  therefore select a persisted specialized method first while retaining canonical coverage for
  every program counter when a runtime guard fails.
- Advance persisted artifact schema and codegen to version 2 and the public runtime code-generation
  facade to ABI v4. The v4 facade exposes stable helpers required by external persisted assemblies
  without granting access to runtime-internal frame or execution-context members.

## Verification and observability

- Cover exact numeric execution, integer overflow, floating division/modulo, negative numeric-for
  steps, Boolean/control semantics, mixed-type guard fallback, stale/malformed profiles, corrupted
  manifests, instruction budgets, Portable PDB output, and the real collectible PE load path.
- Add the `persisted_aot_pgo` benchmark backend. It trains an exact-module profile before emission
  and reports artifact/PDB size, numeric-region count, unboxed local count, direct numeric
  instruction count, and deoptimization statistics. Backend evidence now compares every result
  with the interpreter and adds paired PGO-AOT/Tier-2 thresholds to the six-RID decision.
- Keep Tier-2 profile observations bound to the current closure when pooled scheduler frames are
  reused, so call-heavy workloads export valid exact-module profiles.
- Document profile identity, shared IL, artifact compatibility, safepoints, guards, and canonical
  fallback in ADR 0015.
