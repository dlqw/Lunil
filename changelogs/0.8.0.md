# Lunil 0.8.0

## Stable performance and runtime release

- Ship the qualified reference interpreter, CoreCLR Tier 1/Tier 2 JIT, and guarded loop OSR with
  exact 64-bit instruction accounting, a compact Tier 0 loop, generation-bound direct compiled
  calls, and bounded weak table/operand PICs.
- Remove the Lua persisted/static AOT product, including its public APIs, PE/PDB artifacts,
  loaders, executors, disk caches, `Lunil.Build` package/tasks, CLI/config target, benchmark rows,
  and release gates. Legacy target inputs fail closed with `LUNIL0006`, phase `removed-feature`,
  and exit code `2`.
- Retain .NET NativeAOT/trimming support for the compiler, runtime, hosting, and CLI surfaces.
  Dynamic-code-unavailable processes deterministically use the reference interpreter without
  claiming Lua AOT success.
- Freeze the exact public APIs of 13 assemblies and the metadata, dependency graph, assets,
  symbols, consumers, and tool behavior of 13 NuGet packages for the stable `0.8` line.

## Release artifacts and validation

- Release validation covers six-RID build, test, conformance, NativeAOT, packaging, and
  cross-runtime performance gates.
- Immutable tag `v0.8.0-rc.1` published six RID bundles, 13 NuGet packages, 13 symbol packages,
  GitHub Packages, and the GitHub prerelease.
- Stable `0.8.0` removes the suffix from that product without product-code, public-API,
  package-scope, canonical-IR, JIT-profile, or evidence-schema changes. Its tag repeats the same
  version-derived release gates and publishes a non-prerelease GitHub Release.

## Compatibility

- `0.8.0` is intentionally breaking relative to `0.7.0` because Lua AOT and `Lunil.Build` were
  removed. See the [`0.8.0` migration guide](../docs/migration-0.8.0.md).
- Backward-compatible fixes use `0.8.1`. New public API, backend architecture, IR, and generated
  code work starts at `0.9.0-alpha.1`.
