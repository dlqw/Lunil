# Lunil 0.8.0-alpha.4

## Generational GC and table-key lifetime

- Give every logical heap object a validated dense slot and use swap-removal during sweeping,
  eliminating the previous linear search and list-tail shift for every collected object while
  keeping stable `ObjectId` identity independent from storage order.
- Maintain an exact young-generation set. Minor cycles now whiten, scan, snapshot, sweep, and reset
  young objects only, while force-traversing remembered old owners through the normal strong/weak
  visitation rules. Old-root mutations during an incremental minor cycle are queued immediately.
- Preserve pending finalizers, resurrection, handles, weak values, weak-key ephemerons, promotion,
  and full-cycle reclamation across repeated minor collections with focused regression coverage.
- Store deleted collectable table-key identity as an object ID and clear the tombstone's `LuaValue`,
  preserving `next` and reactivation semantics without retaining the key as a managed CLR root.
- Document stable object identity, mutable dense slots, generation membership, remembered barriers,
  and tombstone invariants in ADR 0014.

The 4,000/8,000-old-object microbenchmark now reports flat minor-GC temporary allocation (about
4.2/4.1 KiB locally) instead of the previous approximately 1.89x growth.
