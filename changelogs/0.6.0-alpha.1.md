# 0.6.0-alpha.1（开发中）

## 新增

- 新增独立 `Lunil.StandardLibrary.Tests` 测试项目，标准库不再只依赖运行时测试间接覆盖。
- 完整实现 Lua 5.4 `math` 模块，包括整数/浮点 tag 保持、边界运算、比较元方法，以及与
  PUC Lua 5.4.8 相同的 xoshiro256** 随机算法和无偏整数区间投影。
- 完整实现 Lua 5.4 `utf8` 模块，包括 strict/lax 解码、扩展 UTF-8、`codes` 泛型迭代器、
  相对位置与非法字节定位。
- 实现 Lua 5.4 basic library，包括受保护 metatable、raw 操作、迭代器、数值/字符串转换、
  warning/GC 控制、可注入 console/filesystem，以及 text/binary/reader `load`、`loadfile` 和 `dofile`。
- 新增标准库宿主能力入口 `LuaStandardLibraryOptions`、`ILuaFileSystem` 与 `ILuaConsole`；默认实现
  使用系统文件与标准输入输出，测试可注入完全内存化 provider。
- 新增 full userdata、light userdata、每 userdata metatable/user values、宿主 payload 生命周期，
  以及每 `LuaState` 独立且永久根化的 registry。
- resumable native step 新增 GC 可见的每调用状态与显式 callback yield 边界，支持标准库函数
  在递归、重入、Lua callback 和 logical GC stress 下保持状态隔离。
- `pcall`/`xpcall` 的 protected boundary 扩展到 resumable native target，覆盖 callback、yield、
  error handler 与 tail-position 返回，不再只支持同步 native 或 Lua closure。
- 新增 Lua 5.4 `table` 模块的全部入口，覆盖带元方法的 length/index/newindex、重叠 `move`、
  pack/unpack、concat、insert/remove、比较 callback 与不可 yield 的 per-invocation 状态。
- 新增 Lua 5.4 `string` 模块第一轮完整 API 面，包括字节字符串操作、显式 Lua pattern VM、
  captures/frontier/balanced/back-reference、`gmatch`、callback/table/template `gsub`、format parser，
  以及大小端、对齐、定长/变长字符串与 1–16 字节整数的 pack/packsize/unpack parser。
- `table.sort` 改为 PUC Lua 5.4.8 `auxsort`/partition 等价的显式 continuation machine，复刻
  comparator、`geti`/`seti` 调用顺序、invalid-order 检测、较小分区递归及失衡 pivot 随机化。
- `string.format` 改为 per-invocation parser 状态机；`%s` 现在使用 `luaL_tolstring` 等价语义，
  支持不可 yield 的 `__tostring`、递归重入、embedded NUL 规则，并补齐整数 precision/alternate、
  `%q` literal、hex-float、格式校验等边界。
- `string.pack` 的 `X` 选项严格消费紧随其后的单个 option，不再越过空格或配置 option；pattern
  bracket class 同步支持首字符 `]`，并新增空匹配、NUL、回溯 capture 与 malformed corpus。
- 新增 canonical IR 到 PUC Lua 5.4 prototype/chunk 的反向 lowering，并接入 `string.dump`；覆盖
  常量、算术/比较元方法 companion、跳转、数值循环、vararg/open call、非 block 对齐 `SetList`、
  嵌套 prototype/upvalue descriptor、debug metadata 与 strip，输出已由 PUC Lua 5.4.8 执行验证。
- 新增 Lua 5.4 `package` 模块与全局 `require`，包括 registry-backed `loaded`/`preload`、四级
  searcher、环境变量/default path 展开、`searchpath`、Lua 文件 loader data/cache 语义，以及稳定的
  unsupported native loader 结果；模块 loader callback 保持 PUC C library 的不可 yield 边界。
- 新增完整 `io` 模块、`FILE*` full userdata 与可替换文件/console/pipe provider，覆盖标准流、全部
  open mode、`read` 的 numeral/all/line/byte formats、write/flush/seek/setvbuf、`io.lines` 自动关闭、
  `popen`/`tmpfile`、`__close`/`__gc`、幂等资源释放及错误三元组。
- 新增完整 `os` 模块与可注入 clock/timezone/process/locale/environment/filesystem mutation provider，
  覆盖 date table/UTC/strftime、time normalization、execute/exit、rename/remove/tmpname；`os.time` 的
  `__index`/`__newindex` 使用 GC 可见、不可 yield 的 per-invocation callback 状态机。
- 新增完整 `debug` 模块及 runtime debug API，覆盖跨 coroutine frame、source line/local range、
  getinfo/getlocal/setlocal、registry/metatable/uservalue、upvalue identity/join、traceback、交互 debug、
  call/return/line/count/tail-call hook 与 hook yield barrier。
- runtime 可为 resumable-native → resumable-native callback 自动插入隐藏 Lua trampoline，解除原先必须
  由调用者手工跨 Lua frame 的限制，并保持 callback continuation、yield、GC rooting 与 debug 栈可见性。
- 新增 `LuaStandardLibrary.InstallAll`，一次安装 Lua 5.4 的全部标准库与全局入口。
- 接入 PUC Lua 5.4.8 官方 portable user-mode 标准库语料，持续执行 `strings.lua`、`pm.lua`、
  `tpack.lua`、`math.lua`、`utf8.lua` 与 `files.lua`；覆盖 pattern/format/pack、随机数、文件 I/O、
  `loadfile`/dump、日期时间、generic-for closing value 以及 debug local 等组合语义。
- CI 扩展为 Windows、Linux、macOS 的 x64/Arm64 六个原生 runner，所有 RID 都执行构建、全量测试、
  官方兼容性语料和 native-platform bundle 生成。
- 新增 callback/GC fuzz 回归，在 every-allocation logical GC 下反复组合 `table.sort`、嵌套排序、
  `string.gsub` callback、不可 yield 边界与临时文件 finalizer。

## 变更

- `LuaValueKind` 增加 `Userdata` 与 `LightUserdata`；`LuaValue` 增加对应构造和访问 API。
- `LuaRuntimeOperations`、`LuaOperationResolution`、`LuaResultTransform` 与
  `LuaValueOperations` 公开，供独立标准库程序集复用唯一的元方法和数值语义实现。
- canonical upvalue metadata 保留 PUC descriptor kind，Lua 5.4 `SETLIST` block 编号转换同步修正为
  每 block 50 个 array slot，保证较大 table constructor 的 binary load/dump 索引一致。
- Lua state 的初始永久对象由 globals/main thread 两个增加为 globals/registry/main thread 三个；
  logical heap 基线配额和对象计数测试已同步更新。
- source lowering 现在生成逐指令行号、函数定义行与精确 local lifetime，供 debug API、traceback 和
  non-strip binary dump 共用；native 调用同时记录当前 C function provenance。
- `package.loaded` 现在由 `InstallAll` 预注册全部已开启标准库，`require("utf8")`、`require("debug")`
  等与 PUC `luaL_openlibs` 行为一致。
- 修正 Lua 数值和字符串边界：NaN 的顺序比较恒为 false、NaN table 查询返回 nil、浮点 modulo 使用
  `fmod` 加符号校正、float→string 保留 `.0` tag、`%p` 使用对象身份且标量返回 `(null)`，并拒绝
  非 Lua 5.4 的 `%F` conversion。
- 修正 `string.rep`/`packsize` 的 `INT_MAX` 边界、pack 数字 option 消费和 `!16` alignment；
  `gsub` 无替换时复用原始长字符串身份。
- 修正文件与加载边界：`read("n")` 使用 PUC 200-byte numeral scanner、长数字及非法前缀消费一致，
  `read` format 按首字符解析，`os.tmpname` 不预先创建文件，`loadfile` 支持 BOM、首行 `#` comment
  与 binary 首字节 mode 判定，系统文件允许并行读写句柄。
- 修正 runtime stack 缩顶时遗留失效引用、批量 native finalizer 排空、generic-for 四个 `(for state)`
  debug local 与 closing value 标记；控制流 line hook 的 then/end/repeat/closure 行序列与 PUC 对齐。

## 兼容性

- `math.randomseed(123, 456)` 的确定性序列已逐值对照 PUC Lua 5.4.8。
- `math.min`/`math.max` 的 `__lt` callback 保持 PUC C library 的不可 yield 边界。
- Lua 5.4 标准库的全部入口、provider 生命周期、portable user-mode PUC 5.4.8 验收语料、GC/callback
  stress 与六 RID CI 已完成。本项目不提供 Lua C API ABI，因此 `package.loadlib` 的 native C module
  loader 继续返回稳定的 unsupported 结果；这不影响纯 Lua 标准库与宿主 provider API 的完整交付。
