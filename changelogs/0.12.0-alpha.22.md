# Lunil 0.12.0-alpha.22

## Added

- Added `LuaPatchResourceKind.HostResource` and stable userdata migration for non-copyable game,
  native, and host resources.
- Added `LuaPatchStableResourceHandle` with exact resource identity, explicit active leases,
  idempotent ownership, and deferred release after in-flight work completes.
- Added `LuaClrBridge.CreateStableResource` so an exact-allowlisted host object can retain CLR member
  access while participating in patch migration.

## Reliability

- `Continue` now retains the previous host-resource userdata identity in the candidate graph and
  restores the old graph on transaction or health rollback without double ownership or disposal.
- `RejectIfActive` observes stable-resource leases. CLR member calls and event subscriptions retain
  automatic leases, preventing concurrent disposal during an admitted host operation.
- Resource-id mismatches, missing state paths, disposed handles, and adapter-less external lifecycle
  actions fail closed before publication.
