# 0.4.0

## Added

- 新增 `Lua54PrototypeConverter`，递归转换 PUC Lua 5.4 prototype 树、常量、upvalue、寄存器窗口、控制流、调试行号与局部变量范围。
- 完整转换全部 Lua 5.4 opcode 类别；关联的 `MMBIN*`/`EXTRAARG`/测试跳转折叠为 canonical 原子语义，`CALL`/`TAILCALL`/`RETURN`/`VARARG`/`SETLIST` 保留开放栈窗口。
- 新增 `LuaState.LoadBinaryChunk` 与 `LuaInterpreter.ExecuteBinaryChunk`，binary chunk 可直接加载为 heap-owned 主闭包并执行。
- 新增 binary chunk 的 PUC Lua 5.4.8 差分、协程状态机 fuzz、GC stress、畸形控制流和数值 tag 双向互操作测试。

## Changed

- canonical IR 格式升级到 v3，增加 binary source name、定义行、逐指令 source line/logical PC、二进制 upvalue 名和 canonical debug-local 范围。
- PUC chunk verifier 现在检查全部寄存器/常量/upvalue/调用窗口、算术与 `MMBIN*` 配对、不可进入的关联指令、开放 top 来源、数值循环/泛型循环目标、行表和 `<close>` 控制流状态。
- numeric-for 状态机采用 PUC 5.4 的无符号剩余计数模型，覆盖完整 64 位整数边界与 float/string 转换规则。
- `NEWTABLE` 的 array/hash 容量提示被保留到运行时分配，并计入 Lua 逻辑 GC 配额。

## Fixed

- 修正 PUC binary chunk 中 integer tag `3` 与 float tag `19` 的读写顺序。
- 修正 tail-position `coroutine.yield`/`resume` 恢复时开放结果窗口丢失。
- 修正负立即数 shift 在 `MMBINI` 中切换 `__shl`/`__shr` 及翻转操作数的语义。
