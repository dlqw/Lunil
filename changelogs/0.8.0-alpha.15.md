# Lunil 0.8.0-alpha.15

## Bounded table and primitive-operand specialization (#58)

- Add exact-kind Tier 2 guards for boolean, nil, and string equality and for string relational
  operations. Guard failure executes the existing generic operation from the same canonical PC.
- Add a four-entry weak table PIC for dense integer keys, interned string fields, and guarded
  metatable absence. Entries validate table/key identity and shape, storage, metatable, or content
  mutation versions without retaining Lua heap owners.
- Perform one integer array/hash probe and allow no-metatable integer set/append to enter the dense
  array directly. An empty table may create the bounded `[nil, value]` representation for key `2`,
  provided that key is not already present in the hash part.
- Publish sampled table PIC hit/miss estimates and exact invalidation counts through JIT statistics
  and both performance schemas. The first hit/miss is exact and steady-state observations are
  aggregated in batches of 256 to avoid an atomic operation on every access.

## Numeric-region table side operations

- Prove local table identity with a CFG forward must-analysis: `NewTable` creates an exact token,
  `Move` propagates it, every register write clears it, and joins retain only tokens shared by all
  predecessors.
- Keep integer/string table accesses inside verified numeric regions when the table token and key
  representation are proven. Induction variables, arithmetic, comparisons, and accumulators remain
  CLR locals; string constants are materialized once per compiled entry.
- Preserve exact instruction accounting by using the existing hot basic-block reservation and cold
  one-instruction tail as the sole budget source. Table guards fail before a set side effect,
  rollback at the table PC, materialize live state, and resume canonical semantics.
- Poll table-bearing regions at most every 256 backedges while retaining write barriers, ownership,
  metatable behavior, GC/finalizer progress, and generation invalidation.
- Delay whole-function Tier 2 publication until one complete Tier 1 invocation has supplied a full
  function profile. Loop OSR remains responsible for taking over a single long first invocation.

## Correctness and performance qualification

- Add differential coverage for integer array/hash probes, leading-nil growth, owner/write-barrier
  behavior, weak string/metatable entries, mutation invalidation, megamorphic bounds, primitive
  equality/ordering, compiler-proven table flow, numeric-region table execution, metamethod deopt,
  exact budgets, reload, and owner release.
- The six-round balanced win-x64 backend record measured `sieve` from a Tier 1 median of
  **9.673 ms** to **1.693 ms** (**5.64x** paired median) and
  `table_access` from **1.152 ms** to **265.880 µs** (**4.41x** paired median). Their six paired
  samples were respectively 4.99x–5.80x and 3.85x–5.39x, above the 1.50x target without guard
  failures or invalidations.
- The same record has an empty `NegativeWorkloadGateFailures` list; table PICs remained bounded and
  target-workload Tier 2 compilation p95 stayed below 3 ms. The aggregate RID decision remains
  unqualified only because the pre-existing arithmetic compile-allocation p95 and slope gates are
  still above their limits; those gates were already failing on the alpha.14 baseline and were not
  relaxed for this release.
