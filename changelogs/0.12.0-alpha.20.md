# Lunil 0.12.0-alpha.20

## Added

- Added the `PatchTable` state migration rule for preserving a live table's identity while adopting
  candidate entries, metatables, weak-table behavior, and compatible function versions.
- Added `MaximumTablePatchEntryCount`, a commit-wide bound shared by nested state-table patches and
  module-level `PatchExistingTable` publication.

## Reliability

- Table patch journals now root previous and candidate keys, values, metatables, and detached
  candidate tables through publication, health checks, and rollback. Full Lua collection can no
  longer discard graph objects required to restore a rejected patch.
- Migration schemas now reject overlapping state target paths, preventing order-dependent writes to
  an owning table and its descendants.
