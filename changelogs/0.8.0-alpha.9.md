# Lunil 0.8.0-alpha.9

## Versioned JIT invalidation

- Add per-module JIT generations and active execution leases. `Invalidate`, `ClearCache`, and
  executor disposal now reject old entries, clear Tier 1/Tier 2/Loop OSR methods, and wait until
  admitted old-generation delegates have exited.
- Make generated entry checks and Loop OSR backedges return `BackendInvalidated` at a canonical
  safepoint when their generation becomes stale. New-generation code may be compiled normally
  without allowing an old delegate to re-enter.

## Immutable function versions and reload migration

- Introduce stable lexical `LuaFunctionIdentity`, immutable `LuaFunctionVersion`, atomic closure
  slots, and frame-entry snapshots. Active and suspended frames keep their captured code and cache
  identity; later calls through the same closure observe a compatible published successor.
- Extend `ReloadModule` results with structured function migration status and counts. Traverse
  exported tables, metatables, and closure-upvalue graphs, preserve closure/upvalue identity on an
  exact layout match, and leave missing or incompatible replacements on old code.
- Keep candidate discovery module-scoped so a shared `_ENV` cannot accidentally select an old
  alias as its own replacement. Debug metadata, constant materialization, table hints, logical GC
  roots, frameless calls, persisted AOT, and static AOT now use the frame snapshot where required.

## Conservative profile remapping

- Add `LuaJitProfileRemapper` and structured `LuaJitProfileRemapResult`. Valid source profiles can
  cross a `ModuleContentId` change only for functions with the same lexical identity, signature,
  upvalue layout, instruction count, and complete canonical `Opcode/A/B/C/D` stream.
- Permit constant-only changes, map dense function-ID shifts through lexical identity, rewrite
  self-module Lua call targets, give changed/added functions empty profiles, and reject malformed
  or wrong-source payloads before import.

## Design and verification

- Record the generation lease, frame snapshot, closure migration, conservative identity, profile
  mapping, rollback limits, and external-watcher boundary in ADR 0017.
- Cover concurrent invalidation, generation safepoints, compatible/incompatible reload, suspended
  coroutines, cross-version PGO mapping, API snapshots, and existing runtime/backend/hosting gates.

This release resolves Issues #38, #40, and #41.
