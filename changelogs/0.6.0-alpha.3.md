# 0.6.0-alpha.3

## 新增

- 新增可独立开启的实验性 loop OSR：natural-loop/backedge 验证、canonical register entry map、
  budget/hook/GC/close entry guard、精确 guard recovery，以及 numeric/generic/while/repeat 循环入口。
- OSR 编译接入 bounded queue、并发去重、retry、tiered code-byte LRU、module/cache invalidation、
  dispose/cancellation 与结构化 request/compile/entry/exit/guard/invalidation 事件和统计。
- Runtime benchmark corpus 新增 interpreter/OSR 同源 steady-state 循环对比；Windows x64 本地中位数显示
  约 47.5% 吞吐提升，但分配量仍从约 1.8 KiB 增至 9.7 KiB，因此 OSR 默认保持关闭。

## 语义与恢复

- OSR 请求仅在 verified backedge 已完成、同一 frame 的 canonical PC 已提交到 loop header 后生效，
  不会在 backedge 副作用前切换。
- entry map 保持 canonical register、frame top、open upvalue 和 to-be-closed state materialized；
  call、yielding metamethod、close handler 与复杂操作继续通过 Runtime ABI/shared scheduler 执行。
- guard failure 从当前 canonical PC 回到 reference executor，已完成的迭代、close 或其他副作用不会重放。

## 验证

- 覆盖 numeric/generic/while/repeat、closure capture、open upvalue、to-be-closed、yielding metamethod、
  instruction budget、exact debug hook、every-allocation GC、并发编译去重、invalidation 和 injected guard failure。
- OSR 保持 explicit opt-in；`EnableLoopOsr = false` 时不请求、不编译且不进入任何 OSR entry。
