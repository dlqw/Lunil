# Lunil 0.7.0-alpha.1

## Release classification

- Start the `0.7.0` alpha series. The `0.6.0` development line ended at the immutable
  `0.6.0-alpha.14` execution-backend preview and will not receive a suffix-free stable release.
- Keep the qualified interpreter, Tier 1/Tier 2 JIT, exact-numeric loop OSR, persisted CIL AOT,
  build-time AOT, and NativeAOT paths as regression-gated execution foundations.
- Define `0.7.0` as the compiler, analysis, hosting, workspace, and CLI product milestone. The
  feature/API scope remains open, so this version is an alpha prerelease rather than beta, RC, or
  stable.

## Compiler product boundary

- Add the `Lunil.Compiler` package and `LuaCompiler` public pipeline for byte source, UTF-8 text,
  logical source identities, bounded lexer/parser/binder options, canonical lowering, independent
  verifier budgets, and immutable compilation results.
- Attribute diagnostics to lexing, parsing, binding, lowering, or verification while deduplicating
  diagnostics propagated between existing compiler layers.
- Preserve logical source names in every canonical function's debug metadata so later tracebacks,
  persisted artifacts, and PDB generation do not depend on checkout-specific absolute paths.
- Check cancellation between each existing compiler phase and never publish a canonical module
  after an error or configured verifier failure.

## Hosting product boundary

- Add the `Lunil.Hosting` package and reusable `LuaHost`, which composes compilation, runtime state,
  standard-library installation, canonical execution, and validated Lua 5.4 binary-chunk execution.
- Add explicit Trusted, Restricted, and Deterministic profiles. Restricted denies file-system,
  environment, and process capabilities and uses a binary-safe buffered console. Deterministic
  additionally fixes UTC/Unix-epoch time, clock, locale, and table hash seed.
- Preserve explicit host overrides for standard-library providers and expose state, heap, stack,
  call-depth, and instruction budgets through the existing Runtime options instead of duplicating
  quota behavior.
- Return structured compile-and-run results that distinguish compile failure from a started Lua
  execution, and preserve globals across executions on the same host-owned state.

## Build and integration

- Route `Lunil.Build` source compilation through `Lunil.Compiler` instead of maintaining a second
  parse/bind/lower assembly path.
- Use stable logical module names for build-time source debug identity and cache/artifact input,
  avoiding checkout-specific absolute path identity.
- Add compiler and hosting projects and their test suites to `Lunil.sln`; packaging and release
  bundle discovery include the new public packages automatically.
- Route the NativeAOT fixture's dynamic-source path through `Lunil.Compiler`, keeping the public
  compiler boundary covered by trimming/AOT analysis rather than validating only internal layers.

## Tests and documentation

- Cover successful source compilation, phase diagnostics, configured verifier rejection,
  cancellation, invalid UTF-8 in byte-oriented comments, compile-and-run, reusable state,
  restricted capabilities, deterministic time/hash behavior, and explicit capability overrides.
- Add the `0.7.0` roadmap with later annotation, type/flow, incremental workspace, CLI, beta, RC,
  stable, and explicit non-goal contracts.
- Update bilingual README examples to use the product-level `LuaHost` boundary while preserving
  lower-level compiler, IR, runtime, JIT, AOT, and NativeAOT documentation.
