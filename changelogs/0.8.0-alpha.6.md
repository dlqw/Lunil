# Lunil 0.8.0-alpha.6

## Runtime hot-path validation and stack access

- Add frame-internal unchecked stack reads and writes after the VM has established register and
  frame bounds. Dynamic result windows still grow explicitly, enforce `MaximumStackSlots`, update
  frame tops, reject foreign-state values, and run the generational write barrier.
- Route open-upvalue reads, writes, and closing through the same validated stack fast path so
  suspended frames retain young captured values during minor collection without repeating public
  index validation on every access.
- Validate a table mutation's key, value, and barrier owner once, then apply the backward barrier
  through a short-lived validated owner token. Stack writes likewise reuse the already extracted GC
  target while preserving the public cross-state and collected-object error semantics.
- Extend runtime regression coverage for foreign native results, dynamic multireturn stack growth
  and stack limits, and old-thread-to-young open-upvalue writes in generational mode.
- Add a mixed-kind `LuaValue.Kind` microbenchmark. Profiling retains the current 16-byte value
  representation: adding a separate kind field would expand each value to 24 bytes while kind
  dispatch is not a leading runtime cost in the measured workloads.
