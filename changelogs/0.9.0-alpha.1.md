# Lunil 0.9.0-alpha.1

## Unified Tier 2 and Loop OSR numeric specialization

- Remove the legacy 857-line Loop OSR-only Reflection.Emit compiler. Tier 2 function-entry regions
  and Loop OSR backedge entries now share natural-loop analysis, reaching-definition type proofs,
  budget planning, numeric emission, and canonical deoptimization through one pass.
- Preserve the independent Loop OSR configuration, telemetry, and benchmark engine for isolation.
  A loop rejected by the shared proof uses the canonical managed program instead of a second,
  wider specialized emitter.
- Keep exact 64-bit accounting, debug/GC/invalidation guards, weak ownership, managed fallback,
  NativeAOT interpreter fallback, and all stable `0.8` Lua semantics unchanged.

## Architecture decisions

- Retain internal compiler interfaces because deterministic lifecycle tests use them for failure,
  cancellation, blocking, guard-failure, invalidation, and code-budget injection; their dispatch is
  on cold tier-up paths rather than per instruction.
- Close the runtime-shape source-generator proposal: build-time generators cannot know runtime
  module/profile/table/call shapes and must not recreate removed Lua AOT. Direct opcode and library
  switches remain handwritten until a bounded generator prototype demonstrates build/startup or
  drift-prevention value without an extra dispatch layer.
- Record the constraints and future evidence requirements in
  [ADR 0022](../docs/adr/0022-unified-numeric-specialization-pass.md).
