# Lunil 0.12.0-alpha.12

## Added

- Added signed forward/rollback intent and canonical, case-sensitive capability requests to patch
  manifests and `lunil patch inspect` output.
- Added host revision classification, explicit verified-signer rollback authorization, and
  capability admission to `LuaPatchAcceptancePolicy`.

## Security

- Rollback admission now fails closed unless the host explicitly authorizes the verified bundle
  signer. Hosts can use a revision ledger to reject a downgrade labelled as a forward update.
- Denied capability requests, intent mismatches, and unauthorized rollbacks are rejected before a
  durable replay reservation is consumed. Capability admission never expands runtime permissions.
