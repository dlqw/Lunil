# Lunil 0.8.0-alpha.5

## Standard-library continuation and I/O allocation reductions

- Preallocate the complete `table.unpack` continuation/result state once, fill callback results in
  place, and copy only the final result window. Integer-boundary ranges now avoid signed overflow
  and terminate correctly at `long.MaxValue`.
- Keep one GC-visible reusable `table.sort` machine state across comparator, indexing, assignment,
  and metamethod callbacks. Return-stack growth is the only point that can replace its backing
  array; all live Lua values are mirrored before every callback so active collections remain safe.
- Add explicitly owned reusable native state to the VM while preserving the snapshot contract of
  the public `LuaNativeStep` factories. `string.format` and `print` use that internal path for
  repeated `__tostring` callbacks.
- Buffer file reads according to `setvbuf` mode and size, bulk-copy buffered spans, and reconcile
  unread/pushback bytes for current-relative seeks and read/write stream transitions. One-byte Lua
  reads now require roughly one underlying stream read per configured buffer instead of one per
  byte; `"no"` mode remains unbuffered.
- Build `string.format` output in the quota-aware pooled native byte buffer, carry it directly over
  callbacks, transfer long output into the final Lua string without a second copy, and emit `%q`
  escapes and width padding without temporary arrays.
- Let `print` borrow immutable Lua string memory, reuse callback argument state, and share its tab
  separator instead of copying every rendered string and allocating a separator per value.
- Extend the runtime microbenchmarks with callback-heavy `table.sort`, `string.format`, and `print` workloads,
  and add regression coverage for GC during sort/unpack/format callbacks, buffered-read transition
  counts, number-parser pushback, seek position, EOF, `setvbuf`, and update-stream writes.
