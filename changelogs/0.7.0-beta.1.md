# Lunil 0.7.0-beta.1

## Feature and API freeze

- Promote the completed compiler, annotation, type/flow analysis, workspace, hosting, CLI,
  runtime, standard-library, and qualified backend product surface to Beta. The planned `0.7.0`
  feature and public-package scope is now frozen; later Beta work is limited to compatibility,
  diagnostics, documentation, reliability, and performance hardening.
- Freeze deterministic generated C# public-API baselines for all 14 shipped assemblies. CI restores
  the pinned generator and rejects additions, removals, signature changes, nullability changes, or
  assembly-scope changes unless the versioned baseline is intentionally reviewed and regenerated.
- Enable the .NET SDK package validator in strict compatible-TFM and compatible-framework modes for
  every packable component. Freeze all 14 NuGet packages by package identity, metadata, dependency
  graph, asset paths, symbol assets, and tool/package type.

## Compatibility and release gates

- Build a clean local-source consumer against every library/MSBuild package and execute the public
  `LuaHost` path. Install the packed `Lunil.Cli` tool, verify its exact version, and execute a source
  smoke test without falling back to a remote package source.
- Make CLI version tests and six-RID conformance evidence derive the active SemVer from
  `Directory.Build.props`, so Beta, RC, and stable promotions retain the same pinned Lua 5.4.8,
  differential-backend, fuzz, soak, and evidence contracts without prerelease-specific constants.
- Require public API and package compatibility checks in normal CI and in the tag-triggered release
  pipeline, in addition to official Lua 5.4.8 conformance, six-RID backend evidence, NativeAOT,
  trimming, ReadyToRun, bundle, formatting, and repository-boundary validation.
