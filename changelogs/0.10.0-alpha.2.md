# Lunil 0.10.0-alpha.2

Lua 5.1 through Lua 5.5 are now explicit compatibility targets while Lua 5.4 remains the default.

- Add the independent Lua 5.1 chunk reader/writer and canonical opcode translation, including
  Lua 5.1's legacy header and number-only constant representation.

- Add an independent Lua 5.2 binary-chunk reader, generated instruction codec, canonical converter,
  and writer for the official PUC Lua 5.2 format.
- Enable Lua 5.2 source compilation and execution with floating-point numeric literal semantics;
  reject Lua 5.3-only floor-division and bitwise operators instead of silently reinterpreting them.
- Add the Lua 5.2 `bit32` library and version-specific standard-library surface filtering.
- Add Lua 5.2 source/chunk differential coverage and PUC oracle configuration through
  `LUNIL_PUC_LUA52` and `LUNIL_PUC_LUAC52`.
- Add explicit Lua 5.5 adapter identity and version-gated chunk boundary while retaining the
  shared canonical execution pipeline.
