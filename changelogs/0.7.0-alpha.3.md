# Lunil 0.7.0-alpha.3

## Semantic type system

- Add the public `Lunil.Analysis` package with immutable semantic models for `any`, `unknown`,
  `never`, nil, literals, unions, intersections, arrays, maps, structural tables, tuples,
  type packs, functions, overloads, callables, generic parameters/instances, classes, aliases,
  and enums.
- Resolve LuaLS/legacy annotation syntax into class, field, alias, enum, generic, operator,
  callable, function, parameter, return, vararg, nullable, optional-field, and structural types.
  Unknown and recursive declarations produce stable diagnostics and deterministic widening.
- Add structural assignability, literal and numeric subtyping, class inheritance, generic
  substitution/inference, function variance, operator lookup, callable overload selection,
  and bounded union normalization.

## Control-flow and constraint analysis

- Build immutable per-function CFGs with entry/exit, branch, loop, break, goto, return,
  predecessor, successor, and reachability information.
- Infer expression, symbol, function, and return-pack types; validate initializers,
  assignments, calls, arguments, operators, indexing, members, and annotated returns.
- Implement nil, `type`, `assert`, short-circuit, literal, and discriminant-field narrowing,
  plus `---@cast`, definite-assignment, unreachable-code, numeric/generic-for, tuple/vararg,
  and loop fixed-point analysis. Inferred mutable table fields widen after assignment instead of
  being incorrectly constrained to their initializer literals.
- Add cancellation and global type, constraint, CFG-block, flow-iteration, generic-
  instantiation, return-pack, and diagnostic budgets. Source `disable`, `enable`, and
  `disable-next-line` directives suppress `LUA6xxx` diagnostics consistently.

## Compiler, packaging, and verification

- Add `LuaCompilerOptions.Analysis`, `LuaCompilationPhase.Analysis`, and immutable
  `LuaCompilationResult.Analysis`; the analysis phase runs after binding and before lowering.
  Warnings preserve runtime execution, while hosts may promote analysis diagnostics to errors
  through the existing module-publication contract.
- Keep types, annotations, constraints, and CFGs erased from canonical IR and runtime behavior.
  Include `Lunil.Analysis` in solution, NuGet, release-bundle, trimming, ReadyToRun, and
  NativeAOT coverage.
- Cover declaration resolution, structural/generic relations, operators/callables, function and
  return inference, narrowing, loops, generic iteration, CFG reachability, goto/break, casts,
  suppression, cancellation, deterministic budgets, recursive types, and malformed annotation
  bytes, plus Compiler/Hosting integration.
