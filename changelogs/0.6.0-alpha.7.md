# 0.6.0-alpha.7

## Tier 1 默认启用

- `LuaJitExecutorOptions.Default.Policy` 与新建 options 的默认 policy 从 `InterpreterOnly`
  迁移为 `Auto`。
- 默认执行仍从解释器开始；只有达到 entry/backedge hotness 且通过确定性收益资格检查的函数才会
  进入有界异步 Tier 1 编译队列。
- `EnableTier2` 默认改为 `false`，`EnableLoopOsr` 继续为 `false`；本次迁移不会默认启用实验性
  profile-guided Tier 2 或 Loop OSR。
- 显式 `InterpreterOnly`、`PreferJit` 与 `RequireJit` 行为保持不变。
- NativeAOT 或其他动态代码不可用环境继续精确回退解释器，不调用 Reflection.Emit。

## 跨平台性能

- 跨平台测量覆盖六个 RID，每个 RID 使用五个独立进程。
- 六 RID arithmetic speedup 中位数范围为 2.428x–2.920x，bootstrap 95% 下界最低为
  2.398x。
- Tier 1 compile p95 最大为 1.586 ms；所有 RID allocation slope 均为 0，negative workload
  gate 均无失败。
- 增加默认 options、热函数晋升、高级 tier 默认关闭与 NativeAOT 默认回退覆盖。

## 兼容性

- 这是有意的默认行为变化。要求完全禁用动态编译的宿主应显式设置
  `Policy = LuaJitPolicy.InterpreterOnly`，或继续使用 `LuaInterpreter`/`LuaExecutor` Tier 0 入口。
- 现有显式 policy 配置不受影响。
