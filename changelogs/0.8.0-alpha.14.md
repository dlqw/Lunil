# Lunil 0.8.0-alpha.14

## Generation-bound direct compiled calls (#64)

- Add a weak, generation-bound direct-call entry for stable fixed-argument/fixed-result integer
  leaves, guarded by module content identity, function ID, closure/module identity, and
  `LuaFunctionVersion.Generation`.
- Execute qualified leaves without materializing a callee frame or returning through the shared
  scheduler, while retaining the ordinary scheduler for debug hooks, GC/finalizers, exact budgets,
  reload/invalidation, coroutine/protected boundaries, varargs, upvalues, and unsupported semantics.
- Preserve unboxed integer caller state across direct calls inside verified numeric regions. Callee
  parameters and results map directly to caller CLR locals, and dynamic callee instruction cost is
  merged into the existing 64-bit region accounting.
- Reset inline instruction/backedge counters at every call entry, and keep hot-quantum, cold-tail,
  GC/finalizer, guard, and invalidation fallbacks distinct so routine safepoints cannot invalidate a
  stable Tier 2 plan.
- Bound code growth to 8 sites and 32 KiB of leaf IL per caller; each leaf is limited to 128
  instructions, 32 registers, 8 parameters, 4 results, and 1024 backedges.

## Lifetime, diagnostics, and coverage

- Reuse caller generation leases across direct leaves. Invalidation waits for active direct calls,
  while eviction, reload, cache clear, disposal, and owner-release remove only weak cache edges.
- Add direct entry/completion/fallback/invalidation and avoided-scheduler-exit statistics to the
  public JIT telemetry and backend/cross-runtime evidence schemas.
- Cover exact-budget restart PCs, target mismatch materialization, long-loop safepoints, active
  invalidation, closure/module owner collection, and site/code-size limits.

## Performance qualification

- The qualified win-x64 record reduced the backend `fib_iter` median from **3.600 ms**
  (`0.8.0-alpha.13`) to **129.710 µs** (**27.76x**),
  with scheduler exits falling from **70,035** to **39** and execution allocation changing from
  2,654.67 to 2,700.00 B/op (**1.017x**). Tier 2 compile p95 was **6.719 ms**.
- Under the same 250 ms/six-round cross-runtime protocol, `function_calls` improved from
  **573.932 µs** to **91.654 µs** (**6.26x**) and reached 0.904x native Lua / 13.809x MoonSharp.
  All results were correct, the Tier 2 route was stable, and timed fallback/deoptimization telemetry
  was clean.
