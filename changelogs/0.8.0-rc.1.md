# Lunil 0.8.0-rc.1

## Release candidate promotion

- Promote the accepted `0.8.0-beta.1` product to its first release candidate without product-code,
  public-API-baseline, package-scope, canonical-IR, JIT-profile, or evidence-schema changes.
- Keep Lua persisted/static AOT removed, including its APIs, artifacts, loaders, executors,
  `Lunil.Build` integration, benchmark rows, and release gates. .NET NativeAOT/trimming remains a
  supported deployment mode with deterministic interpreter fallback when dynamic code is absent.
- Close normal compatibility, diagnostics, documentation, reliability, and performance hardening
  for `0.8`; RC accepts only release-blocking fixes. New backend, API, IR, and generator work moves
  to `0.9.0-alpha.1`.
