# Lunil 0.11.0

Lunil 0.11.0 is the stable capability-controlled CLR interoperation release. The Lua 5.1–5.5
language contracts remain unchanged; CLR access is opt-in and fail-closed.

## CLR interoperation

- Added exact assembly, type, member, delegate, and event allowlists with explicit capability flags.
- Added deterministic construction and member overload resolution, optional/default and named host
  arguments, enum/nullable/numeric/array/tuple conversions, and stable diagnostics.
- Added userdata properties, fields, methods, indexers, operators, static invocation, `ref`/`out`
  result ordering, and exception translation.
- Added Lua-to-CLR delegate conversion, disposable event subscriptions, callback ownership and
  thread policy enforcement.
- Added `Task`/`ValueTask` wrappers, `clr.await`, explicit disposal, and idempotent ownership.
- Interpreter and dynamic JIT paths share one bridge implementation and conversion behavior.

## Compatibility and deployment

- Existing hosts retain their behavior because `LuaClrOptions.Disabled` is the default.
- The bridge searches already-loaded assemblies only and never loads by name or exposes unrestricted
  reflection.
- NativeAOT and trimming consumers must preserve metadata for their exact allowlists; missing
  metadata fails closed with stable bridge diagnostics.

See [CLR interoperation](../docs/clr-interop.md) and [migration notes](../docs/migration-0.11.0.md).
