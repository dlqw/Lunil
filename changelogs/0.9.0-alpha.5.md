# Lunil 0.9.0-alpha.5

## 字符串构建与内存调度

- 优化短字符串与整数拼接的有界缓存，减少重复转换和临时分配，同时继续遵循 Lua GC
  的对象存活状态与二进制字符串语义。
- 优化 `table.concat` 的双片段写入路径，并保持配额错误顺序和可恢复 native continuation 行为。
- 将稳定的字符串-整数/浮点拼接、字符串长度和密集字符串数组写入纳入共享 Tier 2 区域，
  保留操作数顺序、元表检查、写屏障、精确指令预算和写前 deopt 顺序。
- 增加不参与发布总分的字符串拼接、密集数组写入及 `table.concat` 分段基准；分段结果表明
  现有密集字符串 concat 已不再是发布瓶颈，因此未加入额外的批量复制专用路径。

## 性能门禁

- 增加版本化 `0.9.0` 路线图 target policy，六 RID 聚合会对总体、最低平台和每个发布
  workload 执行硬门禁，并在不达标时返回失败。
- 优化 Tier 2 热循环的 backedge 预算缓存和稳定寄存器边界写回；减少重复乘法、kind 写入和
  DynamicMethod 编译分配，不改变 GC、指令预算或 canonical restart 语义。
- 六 RID Auto JIT 几何均值达到原生 Lua 的 `1.688x`、MoonSharp 的 `24.089x`；最低平台为
  `1.103x`，`control_flow` 为 `2.101x`，`string_build` 为 `2.164x`，全部 `0.9.0` 目标通过。
- Tier 2 与 Loop OSR 的最大编译分配 P95 分别为 `243,208 B` 和 `184,168 B`，低于
  `256 KiB` 与 `192 KiB` 门禁；编译时间、启动、热分配 slope、代码大小和 `0.8.0`
  回归门禁同时通过。

## 验证

- 运行时 GC、弱表、finalizer、值语义、协程和嵌套 native callback 回归通过。
- 全部六个发布 RID 的官方 Lua 5.4.8 user-mode、五后端 differential、确定性 fuzz/GC soak、
  NativeAOT、trimming、single-file、ReadyToRun、包消费者、CLI 和公共 API 门禁通过。
- 可复现的 Alpha 5/Beta 资格数据、源码提交和 workflow run 见
  [性能报告](../benchmarks/results/0.9.0-alpha.5-performance.json)与
  [性能文档](../docs/performance.md)。

Alpha 版本仍处于功能冻结前阶段，API 和后端行为可能继续变化。
