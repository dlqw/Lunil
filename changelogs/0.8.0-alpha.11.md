# Lunil 0.8.0-alpha.11

## Bounded Tier 2 lifecycle

- Feed guard failures back into the Tier 2 profile so unstable branch, type, table, and call-site
  assumptions are removed from the next specialization plan instead of being re-emitted unchanged.
- Bound repeated invalidations of an equivalent Tier 2 plan by the configured compilation-attempt
  limit. Permanently unstable plans now fall back to retained plain Tier 1, while profiles that make
  specialization progress may continue promoting.
- Key guarded closure calls by canonical module content and function identity rather than closure
  object identity. Equivalent closures from fresh `LuaState` instances now share compiled code
  without filling a four-entry identity cache, while reloads and different modules remain guarded.
- Replace the backend-generation lease's per-entry monitor with optimistic atomic admission. Module
  invalidation still rejects new entries and waits for admitted delegates, while normal compiled
  execution no longer enters a monitor on every scheduler invocation.

## Verification

- Add concurrent generation-invalidation coverage, including stale-entry rejection, active-delegate
  draining, and recovery after an unmatched exit attempt.
- Add guard-invalidation and fresh-closure regressions proving observable Lua results remain stable,
  progressive profiles reach Tier 2, and an unchanged unstable plan stops recompiling.
- Under the filtered backend harness, `mandelbrot` Tier 2 warm time falls from approximately `874ms`
  to `4.76ms`; managed allocation falls from approximately `6.39MB/op` to `3.9KB/op`, and queued
  Tier 2 compilations fall from `914` to `3`.
