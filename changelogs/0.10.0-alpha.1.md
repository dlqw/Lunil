# Lunil 0.10.0-alpha.1

## Lua language-version contract

This development release establishes the explicit Lua language-version identity used by the
compiler, canonical IR, runtime state, hosting API, workspace, and CLI.

- Lua 5.1 through Lua 5.5 are represented as distinct version identities.
- Lua 5.4 remains the default and the 0.9.x compatibility baseline.
- `--lua-version`, `luaVersion`, and `LUNIL_LUA_VERSION` select the requested contract.
- The selected version is included in canonical IR and JIT module identity data.
- A runtime state rejects modules from a different Lua language contract.
- Existing Lua 5.4 source, chunk, standard-library, interpreter, and JIT behavior remains the
  default and compatibility baseline.
- Selecting an as-yet unimplemented version fails explicitly instead of silently applying Lua 5.4
  semantics.

## Lua 5.3 compatibility slice

- Lua 5.3 source compilation and interpreter/JIT hosting are enabled with Lua 5.3 module identity.
- Lua 5.4-only local attributes are rejected by the Lua 5.3 parser.
- Lua 5.3 standard-library installation omits the Lua 5.4-only `warn` and `coroutine.close`
  surface.
- A separate Lua 5.3 binary-chunk reader, opcode model, and canonical converter import verified
  Lua 5.3 chunks without routing them through the Lua 5.4 format adapter.
- Canonical Lua 5.3 chunk writing and the complete official Lua 5.3 conformance corpus remain
  follow-up work in this development line.
