# 0.6.0-alpha.6

## Tier 1 性能收口

- 增加 Tier 1 编译阶段、执行路径、slow-path、scheduler exit 与分配诊断，并建立六类 backend corpus。
- 修复 JIT registry 热路径每次 `GetOrAdd` 都创建捕获 factory 的线性分配；算术 workload
  从约 7.46 MiB/op 降至约 2 KiB/op。
- 增加 focused EventPipe allocation trace 脚本和编译 verification/emission/allocation 证据。
- Runtime code-generation ABI 提升到 v2；artifact、cache、profile 和 JIT key 同步失效旧 ABI，
  loader 在 assembly load 前拒绝不兼容 artifact。
- Tier 1 直接 lowering primitive unary/binary/bitwise/comparison、numeric-for、upvalue 和 guarded
  `Close`，并在 Tier 2/OSR 关闭时省略逐指令 observation。
- 增加 owner-safe weak verified-plan cache、ABI `MethodInfo` 缓存与 Reflection.Emit startup prepare；
  本机 arithmetic 单函数多进程 compile p95 为 1.636 ms。
- 增加确定性的 `Auto` 收益资格模型、拒绝原因、break-even class、event/statistics 和 public query；
  imported profile 不得绕过最低资格，`PreferJit` 只覆盖收益过滤。
- cancellation 贯穿 planning、liveness、plan verification、Reflection.Emit 和 registry 发布边界；
  取消的 plan 不进入 owner cache，dispose 后不得发布已编译方法。
- 增加 Runtime、CodeGen 和 backend differential 的 20 轮 focused soak；本地共执行 5,100
  个 focused test case，全部通过。
- 增加五进程性能证据、allocation slope、bootstrap interval、per-RID 决策和六 RID 聚合工具；
  win-x64 arithmetic Tier 1 中位加速为 2.407x，95% bootstrap interval 为
  `[2.065x, 3.858x]`，allocation slope 为 0，本 RID 通过资格门槛。

## 默认策略

- 继续保持 `InterpreterOnly`；win-x64 本地证据已通过，但六 RID 独立 runner 证据需在本阶段提交后
  由 CI 采集并再次审核，不能由单机结果替代。
- Tier 2 与 Loop OSR 继续保持显式 opt-in。
