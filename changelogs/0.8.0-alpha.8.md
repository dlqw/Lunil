# Lunil 0.8.0-alpha.8

## Require provenance and module records

- Track every successful `require` as a monotonic, immutable `LuaModuleRecord` containing the
  module name, loader/searcher provenance, rooted loader and loader data, cache value, and optional
  canonical IR. Public queries validate records against the current `package.loaded` entry and
  lazily release stale roots after direct cache eviction or replacement.
- Distinguish preload, built-in Lua file, and custom-searcher loaders without changing Lua's
  returned loader-data behavior. `load`, `loadfile`, and `dofile` remain independent chunk
  operations and do not acquire named module ownership.

## Idle-only manual reload

- Add `LuaHost.ReloadModule` with structured status, compilation/execution evidence, revision,
  compatible-upvalue counts, table-patch counts, and explicit reporting of failed executions whose
  arbitrary Lua or host side effects could not be rolled back.
- Recompile file-backed modules from their recorded or overridden path. Replay retained preload and
  custom-searcher loaders with the original module name and loader data instead of rerunning a
  mutable searcher chain.
- Keep the previous cache visible until candidate execution completes, then select `ReplaceCache`,
  identity-preserving `PatchExistingTable`, or a custom managed cache policy. Compile, execution,
  and policy failures restore the previous cache, module record, revision, table contents, and
  compatible reused upvalue values.
- Serialize execute, binary-execute, reload, and dispose calls made through one `LuaHost`; reject
  reentrant reload while Lua code is active. This v1 idle observation deliberately leaves strict
  cross-executor invalidation, suspended-coroutine migration, old local function references,
  watchers, and cross-version profile mapping to the versioned concurrency design.

## Design and verification

- Document ownership, prepare/execute/commit semantics, rollback limits, and the exclusion of
  concurrent invalidation in ADR 0016.
- Cover JIT file recompilation, cache replacement and identity-preserving patching, custom policy,
  source override, compatible upvalues, preload/custom replay, direct eviction, rooted-handle
  release, compile/runtime/policy rollback, and reentrant busy rejection.

This release resolves Issues #37 and #39.
