# Lunil 0.8.0-alpha.13

## Compact Tier 0 interpreter loop (#55)

- Execute consecutive same-frame reference instructions without returning through the full shared
  scheduler after every `Continue` exit, while retaining one canonical opcode switch.
- Avoid materializing `LuaCompiledExit` for pure compact instructions and reuse a fully validated
  scheduler/frame state across a proven-pure instruction chain; slow operations and safe points
  revalidate all continuation state before proceeding.
- Keep calls, tail calls, returns, budget polls, debug hooks, continuations, coroutine transfers,
  error/close unwind, deoptimization, and backend probes on the shared scheduler boundary.
- Charge one exact instruction-budget unit before every canonical instruction and accumulate the
  existing 64-bit count across a compact batch.
- Access immutable canonical instructions through their supported backing array and pass them by
  readonly reference, avoiding repeated 40-byte instruction copies.
- Advance logical GC and finalizers every 32 idle compact instructions; reaching the allocation-debt
  threshold or entering an incremental cycle restores per-instruction progress, while
  every-allocation stress and pending finalizers force immediate safe points.

## Hot-path and allocation cleanup (#59 H1, #60)

- Collapse frame routing, backedge commits, instruction observation, and Loop OSR observation into
  optional operations on the existing internal executor boundary. This removes three nullable
  single-implementation interface fields and their repeated casts without adding a Runtime-to-JIT
  project cycle.
- Cache a derived dispatchable-debug-hook state on `LuaThread`, reducing the hot early-out while
  preserving updates when hooks are installed, removed, or recursively executing.
- Replace double materialization for forced coroutine results and yielded values, remove the
  `Concat` temporary in protected native completion, and avoid the unconditional initial argument
  snapshot in `ResolveCall`; `__call` expansion still owns its required arrays.
- Retain argument/closure snapshots that cross yield, resume, continuation, or frame-lifetime
  boundaries, and retain checked register arithmetic where overflow is part of resource safety.
- Reuse the existing unchecked `LuaStack` and open-upvalue fast paths; no duplicate stack semantics
  or weakened owner/write-barrier validation is introduced.

## Correctness coverage

- Extend exact instruction-budget coverage across more than two compact safe-point intervals.
- Cover hook enable/disable across a native call, allocation-triggered Lua finalizers inside compact
  execution, active incremental-GC cadence, and the existing same-invocation backedge-to-Tier-1
  handoff oracle.
- Preserve full Runtime, CodeGen.Cil, backend differential, debug-hook, coroutine, GC/finalizer, and
  conformance behavior.

## Performance qualification

- The qualified win-x64 report records a **0.625x** interpreter geomean versus MoonSharp, all 512
  correct results, and all 16 Auto/Tier 2 product gates passing. Current release measurements,
  methodology, and reproduction commands are maintained in `docs/performance.md`.
