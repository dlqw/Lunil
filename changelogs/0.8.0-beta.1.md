# Lunil 0.8.0-beta.1

## Feature, API, and package freeze

- Promote the completed `0.8` interpreter, Tier 1/Tier 2 JIT, and loop-OSR work to Beta. The
  feature, public API, assembly, package, canonical IR, JIT profile, and benchmark-schema scope is
  now frozen; later `0.8` prereleases accept only compatibility, diagnostics, documentation,
  reliability, and performance-convergence fixes.
- Freeze the exact generated public API of all 13 shipped assemblies and the identity, metadata,
  dependency graph, assets, symbols, and consumer behavior of all 13 NuGet packages. The removed
  `Lunil.Build` assembly/package and Lua persisted/static AOT APIs are not part of the frozen scope.
- Retain the qualified execution product as reference interpreter, CoreCLR Tier 1/Tier 2 JIT, and
  guarded Tier 2 loop OSR. Dynamic-code-unavailable and .NET NativeAOT processes deterministically
  use the same compiler/runtime surface with interpreter fallback.

## `0.7.0` to `0.8.0` migration

- Lua persisted/static AOT compilation, PE/PDB artifacts, loaders, executors, static registries,
  disk caches, `Lunil.Build`, and `LunilCompile` MSBuild items were removed during Alpha. Existing
  consumers should compile source or portable Lua 5.4 chunks at runtime with `LuaHost`,
  `LuaInterpreter`, or `LuaJitExecutor`; see the
  [`0.8.0` migration guide](../docs/migration-0.8.0.md).
- Legacy `--target aot`, `buildTarget: "aot"`, and `LUNIL_BUILD_TARGET=aot` inputs continue to fail
  closed with diagnostic `LUNIL0006`, phase `removed-feature`, and exit code `2`; they never claim
  AOT success through an implicit chunk, JIT, or interpreter fallback.
- `.NET NativeAOT` remains supported and is not the removed Lua backend. NativeAOT, trimming,
  single-file, ReadyToRun, package consumers, CLI tools, bundles, conformance, differential, soak,
  and six-RID performance evidence remain release gates.

## Release validation

- Preserve the Lua 5.4.8 conformance and differential oracles, exact 64-bit instruction budgets,
  debug hooks, coroutine/error/close behavior, logical-GC/finalizer progress, reload/invalidation,
  metatable semantics, table ownership, and write barriers across every remaining backend.
- Retain the fail-closed cross-runtime thresholds for Auto and Tier 2 against MoonSharp and the
  existing compile, allocation, fallback, deoptimization, route, and bounded-cache telemetry gates.
  No AOT row or permanently skipped AOT gate remains in the current schemas or workflows.
