# 0.3.0

## 新增

- 统一 Lua 调用、尾调用、受保护调用、`return`/`__close`、协程切换与 native 恢复的 continuation ABI。
- 增加 `Completed`、`CallLua`、`Yielded` 三态 resumable native step ABI；native 可调用 Lua callback 或自行 yield 后按整数 continuation id 恢复。
- 增加显式 activation-stack 协程调度器、`Start`/`Resume`/`Close` 入口和 `New`/`Suspended`/`Running`/`Normal`/`Dead`/`Error` 生命周期。
- 增加完整的 Lua 5.4 `coroutine` 模块：`create`、`resume`、`yield`、`running`、`status`、`isyieldable`、`wrap`、`close`。
- 增加 `Luac.StandardLibrary` 模块注册骨架；本版本只声明完整交付的 coroutine 模块。
- 增加 heap-owned、owner-aware native closure、显式捕获 slots、写屏障及逻辑 GC traversal。
- 对照 PUC Lua 5.4.8 补齐 yield/resume、close、错误协程、GC stress、三千层 nested resume 和确定性状态机 fuzz 验证。

## 修复

- 未捕获的 coroutine 错误保留 frame、open upvalue、continuation 和 `<close>`，直到 `coroutine.close` 才以原始错误展开。
- suspended coroutine close 以 nil error 逆序运行 closer；显式 close 禁止 yield，普通 return closer 仍可 yield/resume。
- coroutine entry、yield/resume window、terminal error、active resumer chain、native continuation 和 wrap 捕获全部纳入逻辑 GC 根图。
- 内部 coroutine ping-pong 复用 activation 和值窗口；20,000 次切换稳态保持固定分配，不按切换次数增长。

## 兼容性

- canonical IR 保持版本 2；协程 yield 继续通过具 `MayYield` effect 的 `Call` 表达。
