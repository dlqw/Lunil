# Lunil 0.12.0-alpha.10

## Added

- Added `LuaPatchFileReplayStore`, a durable target-scoped reservation store for hosts that share a
  local filesystem. Canonical append-only NDJSON audit events use contiguous sequences, a SHA-256
  hash chain, bounded inter-process writer locking, and stable-storage flushes.
- Replay reservations can be rebound after a process restart until commit. Per-reservation
  operating-system leases serialize candidate execution, support rollback compensation, and allow
  the same signed patch to be rolled out to independent target scopes.

## Security

- Reuse of either a patch id or nonce by another patch in the same target scope is rejected
  atomically. Completed identities, corruption, truncated writes, lock contention beyond configured
  timeouts, and entry or byte limit exhaustion fail closed.
