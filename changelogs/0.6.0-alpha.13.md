# Lunil 0.6.0-alpha.13

## JIT / Loop OSR

- Enable qualified Loop OSR by default for CoreCLR release execution. Automatic admission remains
  limited to verified natural loops that pass exact-numeric runtime qualification and install
  `GuardedExactNumericCil`.
- Keep `EnableLoopOsrManagedFallback=false`. Managed canonical-loop compilation and guard widening
  remain separate experimental opt-ins.
- Preserve `EnableLoopOsr=false` as a complete opt-out from analysis, qualification, specialized
  emitter preparation, compilation, installation, and OSR entry.
- Preserve the dynamic-code capability gate. NativeAOT and other dynamic-code-unavailable runtimes do
  not analyze, prepare, or compile Loop OSR even though the release option now defaults to enabled.

## Rollout evidence

- The protected six-RID report qualified win-x64, win-arm64,
  linux-x64, linux-arm64, osx-x64, and osx-arm64.
- The aggregate minimum arithmetic bootstrap 95% lower bound was 4.349x and the minimum paired
  OSR-on/disabled lower bound was 63.933x. Maximum preparation and compilation p95 values were
  1.239 ms and 6.399 ms.
- Every RID used six exactly balanced processes and at least 30 warm operations per row. All negative
  startup and throughput gates passed with zero automatic acceptance, guard failures, and managed
  installations.
- Backend evidence now measures `loop_osr` with the actual release default and creates the paired
  control with an explicit `EnableLoopOsr=false`; standalone Tier 1 and Tier 2 rows disable Loop OSR
  to retain independent baselines.
- The six-process win-x64 release-default record qualified with a 7.941x arithmetic median,
  118.505x over the explicit disabled pair, 0.545 ms preparation p95, 3.099 ms compilation p95, zero
  allocation slope, and no negative acceptance, guard failure, or managed installation.

## Tests, documentation, and release

- Added default compilation, explicit opt-out, negative-profile, dynamic-code capability, and
  NativeAOT default-contract coverage.
- Renamed the compact benchmark scenario to `jit_default_loop_osr_candidate` and made it consume the
  release default rather than a redundant explicit enable.
- Added ADR 0009 and updated the compiler design, cache contract, performance baseline, NativeAOT
  guidance, bilingual README files, and version policy for `0.6.0-alpha.13`.
