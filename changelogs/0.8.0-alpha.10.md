# Lunil 0.8.0-alpha.10

## Allocation-free debug hook dispatch

- Cache the five fixed debug hook event strings as lazy permanent roots in each `LuaState`, so
  repeated `call`, `tail call`, `return`, `line`, and `count` events neither re-encode UTF-8 nor
  depend on weak string-pool retention.
- Pass the two hook arguments through a GC-safe inline pair instead of allocating an argument
  array, copying it through call resolution, and materializing it again for Lua or native hooks.
- Replace `Enum.HasFlag` on the instruction-level hook path with an inlinable bit-mask test. This
  prevents boxing allocations even in unoptimized builds while preserving the existing hook
  scheduling and transfer semantics.

## Verification

- Add a managed-allocation regression that dispatches more than 10,000 native count-hook events
  after warm-up within a fixed non-scaling allocation budget.
- Verify all cached event strings retain identity through a full logical collection, and retain the
  existing runtime, standard-library debug-hook, backend-differential, API, formatting, and package
  gates.

This release completes the remaining actionable P31 item and closes Issue #42 after the earlier
`0.8.0-alpha.2` through `0.8.0-alpha.6` audit and optimization milestones.
