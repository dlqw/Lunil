# Lunil 0.9.0-alpha.4

## Calls and floating-point regions

- Compile stable fixed-shape primitive Lua leaf calls with exact integer, floating-point, and
  Boolean argument and result types.
- Keep primitive caller and callee values in CLR locals across qualified numeric-region calls.
- Reuse closure guards for the lifetime of one compiled invocation while retaining generation,
  hook, finalizer, and target-mismatch fallbacks.
- Charge straight-line leaves with statically verified instruction costs and retain bounded dynamic
  accounting for branches and backedges.
- Aggregate direct-call telemetry through exact per-thread shards without a contended atomic write
  on every successful call.
- Record fixed call argument kinds in JIT profiles. Profile schema 2 is intentionally incompatible
  with earlier Alpha profile payloads.

## Correctness boundaries

- Preserve scheduler fallback for varargs, open results, upvalues, recursive or native calls,
  unstable primitive types, debug hooks, yields, budget exhaustion, and backend invalidation.
- Discard uncommitted primitive locals before every fallback and materialize the caller at the exact
  canonical instruction.
- Keep the existing limits for bound call sites, generated code, parameters, results, instructions,
  registers, and backedges.
