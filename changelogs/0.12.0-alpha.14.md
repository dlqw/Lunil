# Lunil 0.12.0-alpha.14

## Added

- Added per-target rollout lifecycle adapters that stop new traffic, wait for game-server
  quiescence, and restore routing after a committed or rolled-back ring.
- Added production enforcement for missing lifecycle adapters, bounded lifecycle operation
  budgets, per-target lifecycle results, and explicit isolation, quiescence, and restoration
  failure statuses.
- Added the durable `Restoring` deployment-journal phase so interrupted post-publication traffic
  recovery remains discoverable and recoverable after restart.

## Reliability

- Target restoration runs in reverse isolation order and ignores caller cancellation. Restoration
  failure does not report a false rollback: committed module results remain visible and the journal
  stays incomplete until recovery reconciles host and routing state.
