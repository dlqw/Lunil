# Lunil 0.12.0-alpha.7

## Fixed

- Fixed runtime-owned coroutine migration so the `Continue` disposition installs the previous
  thread into the replacement module at the declared state path, preserving coroutine identity and
  suspended execution state across an atomic patch commit.
- Coroutine continuation now uses the existing state-path mutation journal, so a later transaction
  failure restores the candidate path together with the rest of the patch transaction.
