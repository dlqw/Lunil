# 0.2.0

## 新增

- 建立 `LuaGcObject`、`LuaHeap`、`LuaHandle`、对象 owner、永久根、宿主根和跨 `LuaState` 校验。
- 实现 incremental/generational 三色逻辑 GC、forward/back barrier、remembered set、配额、分配债务、安全点和 GC stress。
- 将 `LuaTable` 重构为 array part 加开放寻址 hash，支持 tombstone、rehash、随机 hash seed、删除后 `next` 继续及三类版本号。
- 实现弱键、弱值、PUC 字符串例外、ephemeron fixed point、finalizer separation、安全点执行、warning 和对象复活。
- 实现类型/对象元表及 `__index`、`__newindex`、`__call`、核心运算、比较、长度和拼接的共享调度。
- 实现 Lua 值错误、protected boundary、`pcall`/`xpcall`、handler failure 和可恢复的完整 `__close` 展开。
- canonical IR 格式升级到版本 2，并增加 `SetTop` 清除死亡临时寄存器。
- 增加 PUC Lua 5.4.8 差分、GC stress、table/heap/IR fuzz、finalizer resurrection、tail-call close 和性能分配门槛测试。
- 增加 runtime benchmark harness，并记录 table、numeric-for 和 full logical GC 基线。

## 修复

- 消除每条 `Close` 指令因开放 upvalue LINQ 快照产生的约 144B 分配；10,000 轮空循环从约 1.45MB 降至约 5KiB 固定分配。
- numeric-for 现在转换数字字符串、拒绝零步长，并在 `long` 边界正确终止，不因回绕继续循环。
- 普通算术遵循 Lua 5.4 数字字符串转换规则，位运算保持不转换字符串。
- tail-call 在异步 Lua `__close` 前保存调用状态，在关闭完成后恢复，并转移 protected/finalizer 边界。
- `xpcall` 拒绝非函数 handler；handler 自身失败返回 `error in error handling`。
- canonical IR verifier 对默认数组、null function、越界 parent、任意 opcode 和伪造 CFG 返回诊断而不是崩溃。

## 兼容性

- 目标运行时保持 .NET 10 / C# 14。
- 生产代码未引入 `unsafe` 或 `System.Runtime.CompilerServices.Unsafe`。
- 不提供 Lua C API 或原生 C 模块兼容。
