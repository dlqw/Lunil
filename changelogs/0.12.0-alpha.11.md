# Lunil 0.12.0-alpha.11

## Added

- Added time-aware signing-key lifecycle policies to `LuaPatchEcdsaTrustStore`. P-256 keys can use
  activation, exclusive expiration, and independent revocation instants, including overlapping
  windows for controlled rotation.
- Added `lunil.patch-trust.v1` trust-store files for `patch verify`, `inspect`, `dry-run`, and `diff`,
  allowing bounded multi-key verification with paths relative to the policy file.

## Security

- Patch verification now uses one time snapshot for lifecycle and signature checks, rejects revoked,
  inactive, and retired signing keys with distinct error codes, and does not trust signer-controlled
  manifest timestamps for key lifecycle decisions.
- Trust stores validate and copy complete P-256 public keys when constructed. The CLI rejects unknown
  policy properties, duplicate ids, malformed keys, invalid windows, oversized inputs, and excessive
  key counts.
